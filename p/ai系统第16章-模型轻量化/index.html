AI系统：第16章 模型轻量化

《AI系统：原理与架构》学习笔记 — 深入理解模型压缩四大件：量化、剪枝、知识蒸馏与低秩分解，从原理到工程部署的完整链路

3 minute read

本章概览

第15章介绍了模型小型化——通过设计紧凑结构（SqueezeNet / MobileNet / ShuffleNet 等）让模型天然轻量。但现实中我们面对的常常是已经训练好的庞大模型，能不能在"事后"做减法？

第16章回答的就是这个问题。本章系统性地讲解模型压缩的四大件：量化（Quantization）、剪枝（Pruning）、知识蒸馏（Knowledge Distillation）与低秩分解（Low-Rank Factorization）。其中前三者最为成熟，也是业界落地的"主力军"。

如果说第15章是"生来就瘦"，那第16章就是"胖了以后再减"——后者的适用场景更广，因为大多数 SOTA 模型都是先训练出一个庞大的教师网络，再压缩给端侧推理使用。

本章各节之间的逻辑关系：

  • 16.1 先给出模型压缩的整体框架（为什么要压、四大类方法、应用场景）
  • 16.2～16.4 深入量化（从基本原理到 QAT / PTQ 再到端侧部署）
  • 16.5 讨论剪枝（结构化 vs 非结构化、四种剪枝流程）
  • 16.6 讲解知识蒸馏（知识类型、蒸馏方式、Hinton 经典算法）

16.1 引言：模型压缩全景

为什么要压缩？

随着神经网络模型规模不断膨胀，三个痛点日益突出：

  • 显存不够：大模型参数量动辄数十亿，移动端几 MB 的可用空间根本放不下
  • 速度太慢：推理时延要求毫秒级，庞大的计算图根本跑不动
  • 功耗超标：端侧设备电池有限，大算力意味着高功耗

模型压缩的目标就是在精度、大小、速度三者之间找到最优平衡点。

模型压缩四件套

方法 核心思路 一句话总结
量化 降低参数表示精度（FP32 → INT8/INT4） “用更少比特表示同一个数”
剪枝 删除不重要的参数或结构 “砍掉不干活的神经元”
知识蒸馏 大模型教小模型 “老师把知识浓缩给学生”
低秩分解 大矩阵分解为小矩阵乘积 “用矩阵分解降维”

  • 大模型压缩：BERT → TinyBERT、GPT → 量化版部署
  • 自动驾驶：车载芯片上低时延目标检测

16.2 量化基本原理

16.2.1 量化的核心价值

图16.2.1：将 FP32/FP16 降低为 INT8 甚至 INT4 等低比特表示——核心思想就是用更少的 bits 表达相同的参数信息。

神经网络的推理过程有几个特点让量化成为可能：

  • 参数量大：有充分冗余空间供压缩
  • 计算量大：矩阵乘加占主导，降低精度收益明显
  • 对噪声不敏感：神经网络天生有一定的容错能力，适度的量化噪声不会严重影响精度

量化带来的四大收益：

优势 说明
加速计算 INT8 vs FP32 加速比可达 3× 以上
节省内存 从 32bit 降到 8bit/4bit，存储直接减到 1/4 或 1/8
保持精度 控制好量化程度，精度损失可控制在极小范围
节能减面积 更少的位宽 → 更少的访存 → 更低的功耗和芯片面积

16.2.2 落地挑战

量化并非"免费的午餐"，实际部署时有几个现实问题：

  1. 精度挑战：低比特（如 INT4）表示能力下降明显，小模型比大模型更难量化
  2. 硬件支持差异：不同芯片对低比特指令的支持程度和计算方式不同
  3. Kernel 优化：需要针对特定硬件体系结构做 Kernel 级优化才能充分发挥性能

16.2.3 量化原理

三种量化方法

图16.2.2：QAT、PTQ Static、PTQ Dynamic 三种方法概览。

方法 核心思想 适用场景 精度损失 易用性
QAT（感知量化训练） 训练中模拟量化，让模型"感知"量化误差并适应 目标检测、分割、OCR 等量化敏感任务 极小 需要重新训练
PTQ Static（静态离线量化） 用少量校准数据计算量化参数，一次性转换 图像分类等对量化不敏感任务 较少 较好（无需训练）
PTQ Dynamic（动态离线量化） 仅量化权重，激活值运行时动态量化 BERT 等大模型，访存瓶颈型 一般 好（无数据需求）

一句话选择指南：如果你追求极致精度且愿意花时间重训练 → QAT；如果想快速拿到量化模型且精度容忍度较高 → PTQ Static；如果只在意模型体积缩小 → PTQ Dynamic。

饱和量化 vs 非饱和量化

图16.2.3：左→非饱和量化（|max| 映射到 127）；右→饱和量化（通过 KL 散度找最优阈值 T，±|T| 映射到 ±127）。

  • 非饱和量化：直接取 |max| 映射，简单粗暴，但对离群值敏感
  • 饱和量化：用 KL 散度找最优阈值 T，在阈值外的值"截断"到边界。这也是 TensorRT 采用的方法

对称量化 vs 非对称量化

图16.2.4：左→非对称量化（含零点偏移 Z）；右→对称量化（Z=0 特例）。

量化本质是一个浮点→定点的仿射映射：

\[ Q = \text{round}\left(\frac{R}{S} + Z\right) \]
  • 对称量化：Z=0，range 关于零点对称，实现更简单
  • 非对称量化：Z≠0，能更精确地刻画不均匀分布的激活值，但计算稍复杂

16.3 感知量化训练（QAT）

16.3.1 QAT 整体流程

图16.3.1：预训练模型 → 插入伪量化节点 → 训练/微调（模拟量化噪声）→ 去掉伪量化节点 → 得到 QAT 模型。

QAT 的核心思想是：在训练过程中就让模型"体验"量化带来的精度损失，从而学会适应量化环境。具体流程：

  1. 基于预训练好的模型获取计算图
  2. 对计算图的关键位置插入 FakeQuant（伪量化）节点
  3. 用训练数据进行微调，最小化量化误差
  4. 训练完成后移除 FakeQuant 节点，得到可直接部署的量化模型

16.3.2 伪量化节点（FakeQuant）

图16.3.2：同时对输入和权重插入伪量化算子。FakeQuant 在前向时量化→反量化，模拟量化噪声；反向时通过 STE 近似传递梯度。

FakeQuant 的插入位置一般在：

  • 卷积层前后
  • 全连接层前后
  • 激活函数（如 ReLU）前后

前向传播：FakeQuant 将数据量化到 INT8 → 立即反量化回 FP32，模拟精度损失 反向传播：量化操作不可导，使用 STE（Straight-Through Estimator） 近似——假设量化操作的梯度为 1，允许梯度"直通"

BN 折叠

图16.3.5：左→原始 BN 层；右→BN 折叠（将 BN 参数合并到前一层卷积的权重和偏置中）。

在推理阶段，为提高效率，通常将 Batch Normalization 的参数"折叠"进前一层的权重和偏置：

\[ w_{\text{fold}} = \frac{\gamma \cdot w}{\sqrt{\text{EMA}(\sigma_B^2) + \epsilon}} \]

QAT 训练时需要模拟这种折叠，在折叠后的权重上应用量化。

图16.3.6/16.3.7：BN 折叠后的训练模型 vs BN 折叠感知量化训练模型对比。

常见的算子折叠组合：Conv + BN、Conv + BN + ReLU、Conv + ReLU、Linear + ReLU、BN + ReLU。

16.3.3 PTQ vs QAT

维度 PTQ QAT
速度 较快 较慢（需重训练）
是否需要训练 否 是
对精度控制 较少 更多（量化参数在训练中学习）
即插即用性 是 需重训练

16.4 训练后量化与部署

16.4.1 PTQ 两种方式

动态离线量化（PTQ Dynamic）：

  • 仅量化权重（FP32 → INT8/INT16）
  • 激活值在推理时动态量化
  • 模型体积缩小 2～4×，但加速效果有限

静态离线量化（PTQ Static）：

  • 同时量化权重和激活值
  • 使用少量无标签校准数据，预先计算量化 Scale

图16.4.2：PyTorch 静态离线的四步流程：加载 FP32 模型 → 融合模块 → 插入 Quant/DeQuant 观察器 → 校准计算 Scale → 转换为 INT8 模型。

量化粒度

图16.4.3：逐张量（per-tensor）使用同一个 Scale；逐通道（per-channel）每个通道独立 Scale，效果更好但计算更复杂。

  • 权重量化：逐通道 > 逐张量（精度更好），但部分硬件不支持逐通道
  • 激活量化：一般采用逐张量（per-tensor）

16.4.2 KL 散度校准法

KL 散度校准法是 TensorRT 采用的经典方法，核心思路是找到使量化前后分布差异最小的阈值：

  1. 收集校准数据集（500～1000 张图片）的激活值直方图
  2. 尝试不同阈值，将 FP32 分布量化为 INT8
  3. 计算量化前后分布的 KL 散度
  4. 选择使 KL 散度最小的阈值

16.4.3 端侧量化推理部署

图16.4.4：三种推理结构：(a) FP32 输入→FP32 输出 (b) FP32 输入→INT8 输出 (c) INT8 输入→INT8 输出。

图16.4.5：INT8 卷积推理流程——数据在线量化涉及三种操作：Quantize（量化）、Dequantize（反量化）、Requantize（重量化）。

端侧推理中涉及三种关键操作：

  • 量化（Quantize）：FP32 → INT8
  • 反量化（Dequantize）：INT32（乘加结果）→ FP32（供下一层使用）
  • 重量化（Requantize）：INT32 → INT8（下一层也需要 INT8 输入）

16.5 模型剪枝原理

16.5.1 剪枝概述

模型剪枝的核心是删除网络中"不重要"的参数，让模型变得更稀疏。与量化不同，剪枝直接改变模型结构，而不仅改变数值精度。

结构化 vs 非结构化

图16.5.1：从左到右，剪枝粒度递增——非结构化（单个权重）→ 滤波器级 → 通道级 → 层级。

维度 非结构化剪枝 结构化剪枝
粒度 逐个参数 滤波器/通道/层
稀疏性 不规则的随机稀疏 规则的结构化稀疏
压缩比 高 受结构单元数量限制
硬件加速 需要特定硬件支持 通用硬件可直接加速
精度影响 可控 可能较大

一句话：非结构化剪枝压缩比高但"看起来乱"（硬件不易加速），结构化剪枝更"工整"但压缩比受限。实际部署中结构化剪枝更实用。

16.5.2 四种剪枝流程

训练前剪枝（PBT）

图16.5.2：PBT 在训练前基于随机初始化权重剪枝，然后训练稀疏网络——无预训练开销。

训练中剪枝（PDT）

图16.5.3：PDT 将训练和剪枝融合——训练完成即剪枝完成。

三种实现策略：稀疏正则化（\(L_1\) 约束）、动态稀疏训练（剪枝+再生循环）、评分法。

训练后剪枝（PAT）⭐ 最流行

图16.5.4：预训练 → 剪枝 → 微调重训练 → 评估 → 迭代——这是最经典的剪枝范式。

四步曲：预训练 → 评估重要性 → 剪枝 → 微调重训练（可迭代多次）。

运行时剪枝

根据输入数据动态决定剪哪些——同一个模型，不同输入看到不同的子网络。优势是灵活性极高，但工程实现复杂。

剪枝算法

基于权重大小的贪心法：权重越接近 0 → 越不重要 → 优先剪掉。

图16.5.5/16.5.6：动态剪枝允许参数恢复——通过掩模矩阵 \(T_k\) 控制连接状态，被剪掉的参数有机会"复活"。

滤波器级结构化剪枝：

图16.5.7：删除某个滤波器 → 对应特征图被移除 → 下一层对应滤波器也被裁剪。

图16.5.8：连续卷积剪枝时的两种权重求和策略——蓝色为被移除的特征图，黄色部分权重是否计入下一层求和需要权衡。

16.6 知识蒸馏原理

16.6.1 知识蒸馏概述

知识蒸馏是 Hinton 在 2015 年提出的思想，灵感来自"物理蒸馏"——把混合物加热，沸点低的先蒸发再冷凝，留下精华。

图16.6.1：大的教师模型将知识"蒸馏"传递给小的学生模型——学生用更少的参数达到接近教师的精度。

知识蒸馏系统由三部分组成：

  • 知识：教师模型的输出（logits）、中间层特征、参数间关系等
  • 蒸馏算法：如何将知识从教师传递给学生
  • 师生架构：教师和学生的网络结构设计与交互方式

16.6.2 三种知识类型

图16.6.2：(a) 基于响应的知识 (b) 基于特征的知识 (c) 基于关系的知识。

基于响应的知识 ⭐ 最经典

图16.6.3：让学生模型的输出 logits 模仿教师模型的输出——“老师把答案直接告诉学生”。

本质是让学生模型的输出分布（软标签）逼近教师模型，损失函数为：

\[ \mathcal{L}_{\text{ResD}}(z_t, z_s) = \mathcal{L}_R(z_t, z_s) \]

基于特征的知识

图16.6.4：不仅学最终输出，还学中间层特征——“不仅抄答案，还要模仿解题过程”。

教师和学生模型的特征图形状可能不同，需要通过转换函数 \(\phi_t, \phi_s\) 对齐。

基于关系的知识

图16.6.5：学习样本之间的关系——不是单个样本的输出，而是一组样本之间的相对关系（距离、相似度等）。

16.6.3 三种蒸馏方式

图16.6.6：(a) 离线蒸馏（教师先训好）(b) 在线蒸馏（师生同步训练）(c) 自蒸馏（深层指导浅层，自己教自己）。

方式 教师模型 学生模型 特点
离线蒸馏 预训练好，参数冻结 从头训练 最常用，灵活选择教师
在线蒸馏 与学生同步训练 同步训练 知识融合，适合多任务场景
自蒸馏 模型的深层部分 模型的浅层部分 无需外部教师，自己教自己

16.6.4 Hinton 经典算法解读

Hinton 蒸馏算法的关键创新是温度系数 \(T\)【Temperature】：

\[ q_i = \frac{\exp(z_i / T)}{\sum_j \exp(z_j / T)} \]
  • \(T = 1\)：标准 softmax
  • \(T > 1\)：概率分布更"软"、熵更大，负标签携带的信息被放大

Hinton 的经典例子：一张宝马的图片，被预测为"垃圾箱"的概率虽然极低（\(10^{-9}\)），但被预测为"胡萝卜"的概率更低（\(10^{-12}\)）。这微小的概率差异意味着宝马和垃圾箱"好歹都是人造物体"，这种"暗知识"正是蒸馏要提取的。

损失函数：

\[ \mathcal{L} = \alpha \cdot \mathcal{L}_{\text{soft}}(p^T, q^T) + (1-\alpha) \cdot \mathcal{L}_{\text{hard}}(y, q) \]
  • \(\mathcal{L}_{\text{soft}}\)：学生模仿教师（软目标）
  • \(\mathcal{L}_{\text{hard}}\)：学生逼近真实标签（硬目标）
  • \(\alpha\) 控制两者的权重

训练完成后将 \(T\) 调回 1，学生模型即可用于推理部署。

小结与思考

第16章构建了模型压缩的完整知识体系，核心要点如下：

  • 量化是当前落地最成熟的方案——INT8 量化几乎成为推理标配，QAT 追求精度极限，PTQ 追求"快"和"省"。理解对称/非对称量化、饱和/非饱和阈值选择是掌握量化的关键。

  • 剪枝从"删参数"的角度做减法——结构化剪枝更适合通用硬件加速，训练后剪枝（PAT）是最主流的流程。一个好的剪枝策略需要在"剪什么、剪多少、何时剪"三者之间做权衡。

  • 知识蒸馏是最优雅的方案——它不是简单地"削参数"，而是让大模型把学到的"暗知识"传递给小模型。Hinton 的温度系数 \(T\) 是点睛之笔。

  • 三种方法可以组合使用：先剪枝 → 再量化 → 最后蒸馏微调，叠加使用往往能获得更好的压缩效果。

模型压缩的本质是一场信息压缩的艺术——如何在用更少的比特（量化）、更少的连接（剪枝）、更小的网络（蒸馏）表达相同的"智能"，是这个领域永恒的主题。
