AI系统学习笔记：第23章 分布式并行

May 21, 2026
2 minute read

引言

分布式训练将训练工作量分散到多个工作节点上，显著提高训练速度和模型准确性。PyTorch 2.0 提供了多种分布式训练方式：数据并行（DP）、模型并行（MP） 和 混合并行（Hybrid Parallel），配合设备网格（Device Mesh）、RPC 分布式训练等可扩展组件，满足不同规模模型的训练需求。

三大核心组件：DDP（分布式数据并行）、RPC（通用分布式结构支持）、C10d（细粒度通信控制）。

23.2 数据并行

23.2.1 DP 与 DDP

数据并行（DP） 将数据集分割成若干子集分配到不同 NPU 节点，每个节点运行相同模型副本处理不同数据子集，每轮结束后汇总梯度更新参数。

但 DP 存在局限：Python GIL 限制多线程效率、全局梯度聚合在单一 NPU 造成资源浪费。

分布式数据并行（DDP） 是 DP 的高级形式，核心改进包括：

  • 多进程实现方式，避免 GIL 限制
  • Ring All-Reduce 算法进行高效集合通信
  • 延迟隐藏技术：反向传播与梯度通信重叠
  • 各 NPU 负载均衡

23.2.2 DDP 实现分析

前向传播：DDP 通过 _sync_module_states 在每次传播前同步各进程的 _parameters 和 _buffers，确保模型一致性。

计算通信重叠：反向传播中，梯度一旦局部计算完成就立即进行通信（通过 autograd 钩子触发），无需等待全部计算完成。reducer 将参数梯度组织到多个桶（bucket）中分桶通信。

数据加载：通过 DistributedSampler 按 NPU 索引切分数据，配合 DataLoader 多进程加载，pin_memory=True 时将数据异步拷贝到 NPU 内存。

23.2.3 异步数据并行

同步数据并行 要求所有 NPU 每轮迭代后同步参数（图23.2.2），慢速 NPU 拖累整体效率。

异步数据并行（ADP） 各 NPU 独立计算和更新，快速 NPU 不受慢速 NPU 影响，但需解决梯度更新竞争和模型收敛性问题（图23.2.3）。

23.2.4 弹性数据并行

弹性训练通过动态资源分配应对节点故障。PyTorch 的 TorchElastic 组件包括：

  • Elastic Agent：管理本地工作进程，监控健康并故障恢复
  • Rendezvous：分布式同步原语 + 节点发现，支持 Barrier、排他性、一致性、故障恢复、共享键值存储

23.3 数据并行进阶：FSDP

23.3.1 前置知识

混精度训练：前向/反向使用 FP16 计算（速度快），参数更新使用 FP32（保证精度）。FSDP 中可通过 MixedPrecision 配置。

损失缩放：动态调整缩放因子（如 2^24 起始），检测梯度溢出时减半，防止 FP16 下溢。

内存消耗 = 模型状态（优化器 + 梯度 + 参数 = 16Ψ）+ 剩余状态（激活值 + 临时存储 + 碎片）。

23.3.2 完全分片数据并行 FSDP

FSDP 基于微软的 ZeRO（Zero Redundancy Optimizer）优化方案：

ZeRO-DP 对模型状态分片，三阶段渐进优化：

阶段 别名 分片内容 显存降低 额外通信
阶段1 ZeRO-1 优化器状态 → 1/4 无
阶段2 ZeRO-2 + 梯度 → 1/8 无
阶段3 ZeRO-3 + 参数 → 1/Nd 1.5×

ZeRO-R 优化剩余状态：

  • 分区激活检查点（按 Nm 分区，释放内存）
  • 恒定大小缓冲区（避免临时缓冲区膨胀）
  • 内存碎片整理

ZeRO-Infinity 突破 NPU 内存壁垒，支持万亿参数训练，在 NPU/CPU/NVMe 间智能迁移参数。

23.3.4 ZeRO 通信分析

  • ZeRO-2：Reduce-Scatter(Ψ) + All-Gather(Ψ) = 2Ψ（与标准 DP 相同）
  • ZeRO-3：前向 All-Gather(Ψ) + 反向 All-Gather(Ψ) + Reduce-Scatter(Ψ) = 3Ψ（标准 DP 的 1.5 倍）
  • ZeRO-R：通信开销通常不到标准 MP 的 10%

23.4 张量并行

23.4.1 朴素张量并行

将模型层分配到不同设备，如 ResNet50 前半部分在 NPU:0，后半部分在 NPU:1。缺点是设备利用率低（同时仅一设备工作），比单 NPU 慢约 7%。

23.4.2 高阶张量并行

更细粒度的模型并行，将单层内部参数拆分到不同设备：

  • MatMul 并行：矩阵按列/行切分并行计算（图23.4.1）

  • Transformer 并行：MLP 中 A 列切割、B 行切割，配合 All-Reduce 汇总；Self-Attention 中 Q/K/V 列切割，线性层 B 行切割

  • Embedding 并行：表切分（每个设备存部分嵌入表）和列切分（每个设备存嵌入表部分列）

  • Cross Entropy Loss 并行：在 vocab 维度分片计算，All-Reduce 取全局 max 和 sum，避免将完整 logits 聚集到单一设备

23.4.3 DeviceMesh 实现 TP

PyTorch 通过 ColwiseParallel/RowwiseParallel/SequenceParallel 等 API 配合 DeviceMesh 实现张量并行。

23.5 流水并行

将模型不同层按序分配到不同设备，输入数据切分为多个 Micro-Batch 实现流水线。

Gpipe 流水并行（F-then-B 策略）：前向→反向→参数更新，存在"气泡"空闲时间，通过激活检查点节省内存。

PipeDream 流水并行（1F1B 策略）：完成一个 Micro-Batch 的前向传播后立即进行反向传播，最大化设备利用率。

PipeDream 是异步的，每个 Worker 使用对应的权重版本，需要管理权重一致性问题。

23.6 混合并行

3D 混合并行（DP+PP+TP） 结合三种并行方式：

  • 内存效率：流水线 + 模型并行双重减少内存消耗
  • 计算效率：ZeRO-DP 优化器状态分区 + 拓扑感知映射
  • 通信优化：张量并行（通信最大）优先放在同节点内，流水并行（通信最小）可跨节点

23.7 昇思 MindSpore 并行

MindSpore 提供完整大模型训练方案：

张量重排布（Tensor Redistribution）：自动插入通信算子（如 All-Gather/Concat）解决设备间张量布局不一致。

通信算子自动微分：定义 All-Gather 的反向算子为 Reduce-Scatter，实现全图自动微分。

并行搜索：基于代价模型（同时考虑计算和通信开销），ResNet50 在 8 设备上搜索耗时 < 1s，训练时间减少 ~55%。

函数式算子切分：动态图下通过 @jit 装饰符指定某段以图模式编译执行，支持算子级模型并行。

流水并行：

MoE 并行：Gate 计算并行化 + FFN 均分到各设备，All-to-All 通信路由数据，参数量扩展但计算量增加较少。

异构并行训练：内存密集型算子→CPU，计算密集型算子→加速器，单机 8 卡可训练/推理千亿模型。

本章小结

并行策略 核心思想 优点 适用场景
数据并行（DDP） 数据分片，模型副本 实现简单，扩展性好 模型可放入单设备
FSDP/ZeRO 模型状态分片 显著降低显存 大模型训练
张量并行（TP） 单层参数/计算切分 细粒度并行 超大单层参数
流水并行（PP） 不同层分配到不同设备 通信量小 深层网络
混合并行（3D） DP+PP+TP 组合 最大化资源利用 万亿参数模型

本章系统介绍了从基础数据并行到 3D 混合并行的完整技术栈，以及 MindSpore 在大模型训练上的工程实践，为理解现代分布式训练框架的设计奠定了坚实基础。
