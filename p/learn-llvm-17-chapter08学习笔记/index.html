2. TableGen 语言概述

2.1 为何使用 TableGen

LLVM 后端的大部分描述（寄存器、指令、调用约定、调度模型等）都用 TableGen 编写，以减少重复代码和源码量。

典型应用场景：

用途 示例
后端 寄存器文件、指令助记符与编码、指令选择模式、调度模型
Clang 命令行选项、内部函数、属性
通用 从 .td 文件生成 C++ 片段

2.2 核心概念

  • 记录 (Record)：名称与值的集合，类似 C++ 结构体
  • 类 (Class)：记录的蓝图，可参数化、可继承
  • TableGen 不是编程语言：只用于定义记录，所有计算在构造记录时完成

2.3 工具链

  • llvm-tblgen：LLVM 项目用，包含 LLVM 所需代码生成器
  • clang-tblgen：Clang 项目用
  • tinylang-tblgen：本章自定义工具，用于生成 Token 与关键字过滤器

3.6 multiclass 与 defm

一次定义多条记录：

class Inst<string mnemonic, int opcode> {

}

multiclass InstWithImm<string mnemonic, int opcode> {
  def "" : Inst<mnemonic, opcode>;
  def I  : Inst<!strconcat(mnemonic, "i"), !add(opcode, 1)>;
}

defm ADD : InstWithImm<"add", 0xA0>;
  • def "" 表示第一条记录名由 defm 后的 ADD 决定，即 ADD
  • def I 生成 ADDI
  • 记录名 = defm 名称 + def 名称（空则仅用 defm 名）

3.7 Bang 操作符

以 ! 开头的操作符，用于构造记录时的计算：

操作符 说明 示例
!add(a, b) 加法 !add(opcode, 1)
!strconcat(s1, s2) 字符串连接 !strconcat(mnemonic, "i")
!cond(c1:v1, c2:v2, ...) 条件选择 !cond(!eq(n,0):RegOp, !eq(n,1):ImmOp)
!eq(a, b) 相等 !eq(n, 0)
!or(a, b, ...) 按位或 !or(KEYC99.Val, KEYCXX.Val)
!and(a, b) 按位与 !and(0x1ffffff, ...)
!xor(a, b) 按位异或 !xor(-1, KEYNOMS18.Val)

完整列表见 Bang Operators。

class InstDesc<string name, string suffix, int delta> {
  string Name = name;
  string Suffix = suffix;
  int Delta = delta;
}

def RegOp : InstDesc<"", "", 0>;
def ImmOp : InstDesc<"I", "i", 1>;
def MemOp : InstDesc<"M", "m", 2>;

class getDesc<int n> {
  InstDesc ret = !cond(!eq(n, 0) : RegOp,
                       !eq(n, 1) : ImmOp,
                       !eq(n, 2) : MemOp);
}

multiclass InstWithOps<string mnemonic, int opcode> {
  foreach I = 0-2 in {
    defvar Name = getDesc<I>.ret.Name;
    defvar Suffix = getDesc<I>.ret.Suffix;
    defvar Delta = getDesc<I>.ret.Delta;
    def Name : Inst<mnemonic # Suffix, !add(opcode, Delta)>;
  }
}
  • foreach I = 0-2：值域语法，也可用 0...3
  • #：字符串连接，类似 !strconcat
  • getDesc<I>：通过类“模拟”函数调用，返回匿名记录，再取 .ret

// Base definition the generator knows about.
class Flag<string name, int val> {
    string Name = name;
    int Val = val;
}

class Token {
    string Name;
}

class Tok<string name> : Token {
    let Name = name;
}

class Punctuator<string name, string spelling> : Token {
    let Name = name;
    string Spelling = spelling;
}

class Keyword<string name, list<Flag> flags> : Token {
    let Name = name;
    list<Flag> Flags = flags;
}

class TokenFilter<list<Token> tokens> {
    string FunctionName;
    list<Token> Tokens = tokens;
}
类 用途
Flag 标志名与值，用于生成枚举
Token 基类，仅含 Name
Tok 无固定拼写的标记（如 comment、eof）
Punctuator 标点符，含拼写
Keyword 关键字，含标志列表
TokenFilter 关键字过滤器，含 Token 列表

void TokenAndKeywordFilterEmitter::emitFlagsFragment(
    raw_ostream &OS) {
  OS << "#ifdef GET_TOKEN_FLAGS\n";
  OS << "#undef GET_TOKEN_FLAGS\n";
  for (Record *CC :
       Records.getAllDerivedDefinitions("Flag")) {
    StringRef Name = CC->getValueAsString("Name");
    int64_t Val = CC->getValueAsInt("Val");
    OS << Name << " = " << format_hex(Val, 2) << ",\n";
  }
  OS << "#endif\n";
}
  • getAllDerivedDefinitions("Flag")：获取所有从 Flag 派生的记录
  • getValueAsString / getValueAsInt：按字段名取值

void TokenAndKeywordFilterEmitter::emitKeywordFilter(
    raw_ostream &OS) {
  // ...
  ListInit *TokenFilter = dyn_cast_or_null<ListInit>(
      AllTokenFilter[0]
          ->getValue("Tokens")
          ->getValue());
  // ...
  for (size_t I = 0, E = TokenFilter->size(); I < E; ++I) {
   Record *CC = TokenFilter->getElementAsRecord(I);
   StringRef Name = CC->getValueAsString("Name");
   uint64_t Val = 0;
   ListInit *Flags = nullptr;
   if (RecordVal *F = CC->getValue("Flags"))
      Flags = dyn_cast_or_null<ListInit>(F->getValue());
   if (Flags) {
      for (size_t I = 0, E = Flags->size(); I < E; ++I) {
        Val |= Flags->getElementAsRecord(I)->getValueAsInt("Val");
      }
   }
   Table.emplace_back(Name, Val);
  }
  llvm::sort(Table.begin(), Table.end(), ...);
  // 生成 std::lower_bound 二分查找代码
  • 从 TokenFilter 的 Tokens 字段取 ListInit
  • 遍历每个 Keyword，合并其 Flags 的 Val 得到标志值
  • 按关键字名排序，生成 std::lower_bound 二分查找

5.1 性能说明

  • 二分查找不一定比 llvm::StringMap 更快
  • 更优方案：生成最小完美哈希（如 gperf、Czech-Havas-Majewski、PTHash）

6. TableGen 的缺点

缺点 说明
计算能力有限 基于简单概念，无法像通用 DSL 那样编程
语义隐藏在后端 自定义后端可改变语义，他人难以理解 .td
后端实现复杂 复杂任务下后端代码量大，维护成本高

7. 关键 API 速查

API 用途
Records.getAllDerivedDefinitions("ClassName") 获取某类所有派生记录
Records.getAllDerivedDefinitionsIfDefined("ClassName") 同上，类未定义时返回空
Record::getValueAsString("Field") 取字符串字段
Record::getValueAsInt("Field") 取整数字段
Record::isSubClassOf("ClassName") 判断是否为某类子类
RecordVal::getValue() 取 Init（可 cast 为 ListInit 等）
ListInit::getElementAsRecord(I) 取列表中第 I 个元素为 Record
emitSourceFileHeader(title, OS) 生成文件头注释
TableGenMain(argv0, &Main) 解析 .td 并调用 Main

8. 总结

  1. TableGen：LLVM 的 DSL，用于定义记录并生成 C++ 等代码，减少手写量。
  2. 语法：class、def、defm、multiclass、let、foreach、Bang 操作符。
  3. Chapter08：用 TableGen 定义 Token/Keyword/TokenFilter，后端生成 TokenKind、lookupKeyword 等，替代 TokenKinds.def 式数据库。
  4. 后端：基于 RecordKeeper 遍历记录，按需生成宏保护的 C++ 片段。
