2. LLVM 通道管理器

2.1 通道的作用与分类

LLVM 将 IR 优化和代码生成分解为独立的通道 (Pass)，通道管理器负责以正确顺序执行这些通道。

通道按作用范围分类：

类型 作用范围 典型用途
模块通道 (Module Pass) 整个 Module 过程间优化、全局分析
调用图通道 (CGSCC Pass) 调用图强连通分量 自底向上遍历、内联决策
函数通道 (Function Pass) 单个函数 过程内优化
循环通道 (Loop Pass) 函数内循环 循环优化

2.2 通道管理器的职责

  • 分析结果共享：跟踪通道依赖的分析、缓存状态，避免重复计算，及时释放内存
  • 流水线执行：按流水线顺序执行，函数通道通常按「每个函数依次跑完所有函数通道」的方式执行，以改善缓存局部性

2.3 为何不硬编码通道顺序？

  • 不同优化级别（O0/O1/O2/O3/Os/Oz）需要不同通道组合
  • 用户可能希望添加自定义通道（如语言特定优化）
  • 通道之间可能有依赖，需要灵活编排（如死代码删除应在产生死代码的通道之后运行）

3. 实现 PPProfiler 通道

3.1 通道目标

PPProfiler（“穷人"分析器）在以下位置插入调用：

  • 函数入口：调用 __ppp_enter(FnName)
  • 每个返回指令前：调用 __ppp_exit(FnName)

用于统计函数调用次数和粗略执行时间。

void PPProfilerIRPass::instrument(llvm::Function &F,
                                  Function *EnterFn,
                                  Function *ExitFn) {
  ++NumOfFunc;

  // 1. 在入口块首条指令处插入
  IRBuilder<> Builder(&*F.getEntryBlock().begin());

  // 2. 创建函数名全局常量
  GlobalVariable *FnName = Builder.CreateGlobalString(F.getName());

  // 3. 在入口插入 __ppp_enter(FnName)
  Builder.CreateCall(EnterFn->getFunctionType(), EnterFn, {FnName});

  // 4. 遍历所有 Ret 指令，在每条 ret 前插入 __ppp_exit(FnName)
  for (BasicBlock &BB : F) {
    for (Instruction &Inst : BB) {
      if (Inst.getOpcode() == Instruction::Ret) {
        Builder.SetInsertPoint(&Inst);
        Builder.CreateCall(ExitFn->getFunctionType(), ExitFn, {FnName});
      }
    }
  }
}

要点：

  • SetInsertPoint(&Inst) 在指定指令之前插入
  • CreateGlobalString() 生成 private unnamed_addr constant [N x i8] 常量

PreservedAnalyses PPProfilerIRPass::run(Module &M, ModuleAnalysisManager &AM) {
  // 避免检测 runtime 自身，防止无限递归
  if (M.getFunction("__ppp_enter") || M.getFunction("__ppp_exit"))
    return PreservedAnalyses::all();

  // 声明 void __ppp_enter(ptr), void __ppp_exit(ptr)
  Type *VoidTy = Type::getVoidTy(M.getContext());
  PointerType *PtrTy = PointerType::getUnqual(M.getContext());
  FunctionType *EnterExitFty = FunctionType::get(VoidTy, {PtrTy}, false);
  Function *EnterFn = Function::Create(EnterExitFty, GlobalValue::ExternalLinkage,
                                       "__ppp_enter", M);
  Function *ExitFn = Function::Create(EnterExitFty, GlobalValue::ExternalLinkage,
                                      "__ppp_exit", M);

  for (auto &F : M.functions()) {
    if (!F.isDeclaration() && F.hasName())
      instrument(F, EnterFn, ExitFn);
  }
  return PreservedAnalyses::none();  // 不保留任何分析
}
  • 跳过声明、无名函数
  • PreservedAnalyses::none() 表示所有分析可能失效，保守但安全

void RegisterCB(PassBuilder &PB) {
  // 1. 流水线解析：支持 --passes="ppprofiler"
  PB.registerPipelineParsingCallback(
      [](StringRef Name, ModulePassManager &MPM,
         ArrayRef<PassBuilder::PipelineElement>) {
        if (Name == "ppprofiler") {
          MPM.addPass(PPProfilerIRPass());
          return true;
        }
        return false;
      });
  // 2. 扩展点：在流水线开头自动添加
  PB.registerPipelineStartEPCallback(
      [](ModulePassManager &PM, OptimizationLevel Level) {
        PM.addPass(PPProfilerIRPass());
      });
}

llvm::PassPluginLibraryInfo getPPProfilerPluginInfo() {
  return {LLVM_PLUGIN_API_VERSION, "PPProfiler", "v0.1", RegisterCB};
}

#ifndef LLVM_PPPROFILER_LINK_INTO_TOOLS
extern "C" LLVM_ATTRIBUTE_WEAK ::llvm::PassPluginLibraryInfo
llvmGetPassPluginInfo() {
  return getPPProfilerPluginInfo();
}
#endif
  • LLVM_ATTRIBUTE_WEAK：静态链接时避免重复定义
  • LLVM_PPPROFILER_LINK_INTO_TOOLS：静态链接到工具时定义，不导出 llvmGetPassPluginInfo

4.2 LLVM 树内插件（ppprofiler-intree-plugin）

  • 在 llvm/lib/Transforms/PPProfiler/ 创建目录
  • CMakeLists.txt：add_llvm_pass_plugin(PPProfiler PPProfiler.cpp)
  • 在 llvm/lib/Transforms/CMakeLists.txt 中 add_subdirectory(PPProfiler)
  • 构建后得到 PPProfiler.so，与独立插件用法相同
  • 静态链接：-DLLVM_PPPROFILER_LINK_INTO_TOOLS=ON，会写入 Extension.def

4.3 LLVM 树内注册表集成（ppprofiler-intree）

需要将通道加入 Pass 注册表，供所有 LLVM 工具使用：

void __ppp_enter(const char *FnName) {
  init();
  Time T = get_time();
  void *Frame = __builtin_frame_address(1);
  fprintf(FileFD, "enter|%s|%llu|%p\n", FnName, T, Frame);
}

void __ppp_exit(const char *FnName) {
  init();
  Time T = get_time();
  void *Frame = __builtin_frame_address(1);
  fprintf(FileFD, "exit|%s|%llu|%p\n", FnName, T, Frame);
}

注意：书中示例 cleanup() 中 if (FileFD == NULL) 应为 if (FileFD != NULL)，否则不会正确关闭文件。

局限性：

  • 非线程安全（单文件描述符）
  • 未检查 I/O 返回值
  • 时间戳包含 I/O 开销，不宜作为精确性能指标；调用次数较可靠

6.5 O0 与扩展点

O0 时默认流水线为空，registerPipelineStartEPCallback 等扩展点不会被调用。若要在 O0 也运行扩展点通道，可调用 runRegisteredEPCallbacks() 手动执行回调。

8. 总结

  1. 通道管理器：负责通道顺序、分析共享与流水线执行。
  2. PPProfiler：模块级插装通道，在入口和返回前插入 __ppp_enter/__ppp_exit。
  3. 集成方式：独立插件、树内插件、树内注册表，按需求选择。
  4. tinylang：通过 PassBuilder 支持 --passes、--load-pass-plugin、-O，并区分新 PM（优化）与旧 PM（代码生成）。
  5. 扩展点：在流水线构造时注入自定义通道，无需修改默认流水线描述。
