AI系统：第15章 模型小型化

《AI系统：原理与架构》学习笔记 — 深入理解模型小型化技术：CNN轻量化（SqueezeNet/ShuffleNet/MobileNet系列）与Transformer小型化（MobileViT/MobileFormer/EfficientFormer）

4 minute read

本章概览

随着深度学习的发展，神经网络被广泛应用于各种领域。模型性能提高的同时也引入了巨大的参数量和计算量。由于大部分深度神经网络模型参数量很大，无法满足直接部署到移动端的条件，因此在不严重影响模型性能的前提下对模型进行重新设计，来减少网络参数量和计算复杂度，是当前热门的研究方向。

第15章从三个层面系统讲解模型小型化：第一，理解模型复杂度的衡量指标（参数量、FLOPs、MACs等）；第二，深入分析CNN模型小型化的七大系列经典架构（SqueezeNet / ShuffleNet / MobileNet / ESPNet / EfficientNet / GhostNet）；第三，探索Transformer模型的小型化方案（MobileViT / MobileFormer / EfficientFormer）。

核心洞察：模型小型化 = 结构重新设计 + 高效算子替换 + NAS自动搜索。它不是简单的"缩小"，而是在精度与效率之间找到最优平衡点的系统工程。

15.1 推理参数基础

15.1.1 参数量与性能的博弈

模型性能提升的同时也引入了巨大的参数量和计算量。一般来说，模型参数量越大，精度越高，性能越好，但同时部署成本也急剧上升。

图15.1.1：左图展示了参数量与模型精度的正相关关系；右图展示了历年AI模型参数量的指数级增长趋势（Sevilla et al., 2022）。

15.1.2 复杂度分析

模型参数量和计算量是两个重要的考量因素：

指标 含义 类比
Params（参数量） 模型中的参数数量 空间复杂度
FLOPs（浮点运算次数） 模型执行所需的总浮点运算量 时间复杂度
FLOPS（每秒浮点运算次数） 硬件计算能力指标 计算速度
MACs（乘加操作次数） 乘法和加法操作的总次数 更贴近硬件的计算量
MAC（内存访问代价） 内存访问的开销 访存成本
内存带宽 数据从vRAM到计算核心的速度 数据传输瓶颈

内存带宽是比计算速度更具代表性的性能指标。它取决于内存和计算核心之间数据传输速度以及总线中并行链路的数量。

15.1.3 典型结构参数量与FLOPs对比

标准卷积层：

  • Params = \(k_h \times k_w \times C_{in} \times C_{out}\)
  • FLOPs = \(k_h \times k_w \times C_{in} \times C_{out} \times H \times W\)

分组卷积：

  • Params = \(k_h \times k_w \times C_{in} \times C_{out} / g\)
  • FLOPs = \(k_h \times k_w \times C_{in} \times C_{out} \times H \times W / g\)

其中 \(g\) 为分组数量，分组卷积可将参数量和计算量降低到原来的 \(1/g\)。

深度可分离卷积：进一步将标准卷积分解为逐通道卷积 + 逐点卷积，大幅降低计算成本。

全连接层：

  • Params = \(C_{in} \times C_{out}\)
  • FLOPs = \(C_{in} \times C_{out}\)

15.2 CNN 模型小型化

神经网络模型被广泛应用于工业领域，但由于存储空间以及算力的限制，大而复杂的模型难以被应用。本章介绍了CNN小型化的七大经典系列。

15.2.1 SqueezeNet 系列

SqueezeNet

SqueezeNet 发表于 ICLR 2017，核心贡献在于提出了 Fire Module：先使用 1×1 卷积降低通道数目（squeeze），然后用 1×1 和 3×3 卷积提升通道数（expand）。它达到了 AlexNet 相同的精度，却只用了 1/50 的参数量。

图15.2.1：Fire Module — squeeze层用1×1卷积降维，expand层混合1×1和3×3卷积升维。

图15.2.2：以卷积层开始，后接8个Fire Module，最后以卷积层结束。

三种设计策略：

  • 将 3×3 卷积替换为 1×1 卷积，参数量减少为 1/9
  • 减少 3×3 卷积的输入/输出通道数
  • 将下采样操作延后，保留更大的激活图

SqueezeNext

SqueezeNext 基于残差结构并使用了分离卷积，核心改进包括：

  • 两阶段瓶颈模块：两个 1×1 卷积将通道数降到原来的 1/4，显著减少参数
  • 低秩分解：使用 CP 分解将 \(K \times K\) 卷积参数量从 \(K^2\) 降为 \(2K\)
  • 3×3 拆分为 3×1 + 1×3：参数量从 9 降至 6，减少 1/3

图15.2.3：输入通道C → 压缩层1(降为1/2C) → 压缩层2(降为1/4C) → 分离3×3层 → 逐点卷积升维 → 输出。

图15.2.4：SqueezeNext Block核心 = 两个1×1卷积减少参数 + 3×3卷积拆成1×3和3×1组合。

15.2.2 ShuffleNet 系列

ShuffleNet V1

ShuffleNet 发表于 CVPR 2018，针对极小网络的计算效率问题，提出两个核心操作：

  1. Point-Wise 分组卷积：降低 1×1 卷积的计算量
  2. Channel Shuffle：解决分组卷积导致的不同组之间信息隔离问题

图15.2.5：(a) 分组卷积导致输出通道只接收部分输入的通道信息；(b)(c) Channel Shuffle将每组再细分后重新混合，实现跨组信息交互。

图15.2.6：基于残差块和Channel Shuffle设计的ShuffleNet Unit，由深度卷积、逐点分组卷积组成。

ShuffleNet V2

ShuffleNet V2 在 V1 基础上提出了 Channel Split 操作，并总结出四条高效网络设计准则：

准则 说明
G1 输入输出通道数相等时，MAC最小，模型最快
G2 分组数增大时MAC也会增大，应针对硬件合理设计分组数
G3 网络碎片化程度越高，速度越慢
G4 不要过多使用逐点运算（Element-wise操作）

图15.2.7：(c)(d) Channel Split — 每个单元开始将通道拆分为2个分支，一个恒等映射，一个经多层卷积，最后通道级拼接（Concat），V2的1×1不再使用分组卷积。

15.2.3 MobileNet 系列

MobileNet 系列是移动端部署最经典的轻量化模型族，涵盖 V1 / V2 / V3 三个版本。

MobileNet V1：深度可分离卷积

核心贡献是 深度可分离卷积（Depthwise Separable Convolution），将标准卷积分解为两步：

逐通道卷积（Depthwise Convolution）：一个卷积核只负责一个通道，特征图通道数与输入完全一致。

图15.2.8：每个卷积核仅处理一个输入通道，不进行跨通道的信息融合。

逐点卷积（Pointwise Convolution）：使用 1×1 卷积将逐通道卷积的结果在通道方向进行加权组合。

图15.2.9：1×1卷积核将上一步的特征图在通道方向进行加权组合，生成新的特征图。

MBconv 模块：由逐通道卷积 + BN + ReLU 组成，通过不断堆叠构建完整网络。

图15.2.10：左为标准卷积（3×3 Conv + BN + ReLU），右为MBconv（逐通道卷积 + BN + ReLU + 逐点卷积 + BN + ReLU）。

此外，MobileNet V1 还引入了两个超参数：

  • 宽度系数 \(\alpha\)（0~1）：控制每层通道数的缩放比例
  • 分辨率系数 \(\rho\)（0~1）：控制输入图像分辨率的缩放比例

MobileNet V2：反向残差与线性瓶颈

MobileNet V2（2018）在 V1 基础上引入了两个核心概念：

反向残差（Inverted Residual）：与传统残差"先降维再升维"相反，先将低维特征扩展到高维，用深度卷积过滤，再降维回去——两端的通道数都很小，整体计算量不大。

图15.2.11：先1×1升维 → 3×3深度卷积 → 1×1降维，与传统残差块方向相反。

线性瓶颈（Linear Bottleneck）：在降维层（最后一个1×1卷积）后不使用ReLU激活，而是线性输出，以避免非线性破坏低维空间的信息。

图15.2.12：输入低维压缩表示 → 1×1升维(ReLU6) → 3×3深度卷积(ReLU6) → 1×1降维(Linear)。

关于 ReLU6：MobileNet 使用 ReLU6（限制最大输出为 6），这是为了在移动端 FP16/INT8 低精度场景下保持良好的数值分辨率，避免激活值范围过大导致精度损失。

MobileNet V3：NAS + 手工优化

MobileNet V3 主要贡献：

  • NAS 搜索：由资源受限的 NAS（Block-Wise）执行模块级搜索，NetAdapt（Layer-Wise）执行局部微调
  • SE 结构（Squeeze-and-Excitation）：全局池化 → 两个全连接层 → H-Sigmoid 获取通道权重

图15.2.13：通过全局池化压缩 → FC → ReLU → FC → H-Sigmoid 为每个通道生成注意力权重。

  • 重新设计耗时层：减少第一个卷积层的卷积核数（32→16），精简最后阶段（Last Stage），减少 11% 运算时间和 3000 万次乘加操作

图15.2.14：左为原结构，右为精简后的高效结构 — 将平均池化层前移，移除多余的卷积层。

  • H-Swish 激活函数：\(H\text{-}Swish[x] = x \cdot \frac{\text{ReLU6}(x+3)}{6}\)，相比 Swish（\(x \cdot \sigma(x)\)），避免了昂贵的 Sigmoid 计算

15.2.4 ESPNet 系列

ESPNet V1：高效空间金字塔卷积

ESPNet 应用在高分辨率图像语义分割，核心贡献是 ESP 模块（Efficient Spatial Pyramid）：将标准卷积分解为逐点卷积（降维）+ 空洞卷积金字塔（多尺度特征提取），大幅减少参数和内存。

图15.2.15：(a) ESP模块 — 逐点卷积降维后，使用K组不同膨胀率的空洞卷积提取多尺度特征。

HFF（Hierarchical Feature Fusion）：空洞卷积的简单拼接会引入网格伪像（棋盘效应）。HFF 通过将不同膨胀率输出的特征图在拼接前进行层次化相加，有效消除了网格伪影。

图15.2.16：(a) 膨胀率r=2的3×3卷积产生的网格伪像；(b) 有无HFF的特征图可视化对比。

ESPNet V2：EESP模块

EESP 模块相比 ESP 模块的改进：输入层采用分组卷积取代标准空洞卷积，结合深度可分离卷积，使得参数量仅为 ESP 的 1/8（当 M=240, g=K=4, d=60 时）。

图15.2.17：(a) ESP模块；(b) EESP模块 — 分组逐点卷积 + 深度可分离扩张卷积；(c) 等价高效实现 — K个逐点卷积合并为单个分组卷积。

15.2.5 EfficientNet 系列

EfficientNet V1：复合缩放

EfficientNet 提出了 复合缩放方法（Compound Scaling Method），通过一个复合系数 \(\phi\) 统一缩放网络的深度（d）、宽度（w）和分辨率（r）：

\[d = \alpha^\phi,\quad w = \beta^\phi,\quad r = \gamma^\phi\]

并满足约束 \(\alpha \cdot \beta^2 \cdot \gamma^2 \approx 2\)，使得总 FLOPs 约增加 \(2^\phi\) 倍。

图15.2.18：(a) Baseline；(b) 仅宽度缩放；(c) 仅深度缩放；(d) 仅分辨率缩放；(e) EfficientNet的复合缩放——同时平衡三个维度。

EfficientNet V2

EfficientNet V2 采用训练感知 NAS联合优化精度、参数效率和训练效率：

  • 除 MBConv 外引入 Fused-MBConv 模块加速训练
  • 使用较小的扩展率（expansion ratio），减少内存访问量
  • 倾向选择 3×3 卷积核并增加多个卷积以提升感受野
  • 移除最后一个步长为1的阶段以减少参数和内存访问

奖励函数：\(A \cdot S^w \cdot P^v\)（A=精度, S=归一化训练时长, P=参数量, w=-0.07, v=-0.05）

15.2.6 GhostNet 系列

GhostNet V1：廉价操作生成冗余特征

GhostNet 的核心思想是 Ghost Module：先通过普通卷积生成一部分"原始"特征图，再通过**廉价线性变换（深度卷积）**生成更多"Ghost"特征图，最后将两者拼接。这样以很小的代价生成了与普通卷积等量的特征图。

在 ImageNet 分类任务中，GhostNet 在相似计算量下 Top-1 达 75.7%，高于 MobileNet V3 的 75.2%。

图15.2.19：(a) 普通卷积直接生成所有特征图；(b) Ghost Module — 先通过普通卷积生成少量原始特征图，再用廉价操作（深度卷积）生成冗余特征图，最后拼接。

GhostNet V2：解耦全连接注意力

GhostNet V2 在 Ghost 模块基础上添加了 DFC（Decoupled Fully Connected）注意力机制，用固定权重的全连接层生成具有全局感受野的注意力图。相比自注意力机制，DFC 更简单、部署效率更高。在 167 MFLOPs 下 Top-1 达 75.3%，显著高于 GhostNet V1（74.5%）。

CNN小型化小结

系列 核心思路 代表年份 关键技术
SqueezeNet Fire Module（squeeze + expand） 2016-2017 1×1卷积降维、低秩分解
ShuffleNet Channel Shuffle + 分组卷积 2017-2018 逐点分组卷积、Channel Split
MobileNet 深度可分离卷积 2017-2019 反向残差、线性瓶颈、SE、NAS
ESPNet 空洞卷积金字塔 2018 ESP/EESP模块、HFF消除网格伪影
EfficientNet 复合缩放 + NAS 2019-2020 深度/宽度/分辨率联合缩放
GhostNet 廉价操作生成冗余特征 2020-2022 Ghost Module、DFC注意力

人工设计（SqueezeNet / ShuffleNet / ESPNet / GhostNet）和 NAS 搜索（MobileNet V3 / EfficientNet）是两条互补的技术路线，共同推动了 CNN 轻量化的发展。

15.3 Transformer 模型小型化

自 Vision Transformer 出现后，Transformer 在计算机视觉领域取得了显著成功。但基于 Transformer 的模型通常具有数十亿甚至数百亿个参数，使其难以部署到资源受限场景。本节介绍 Transformer 的轻量化方案。

15.3.1 MobileViT 系列

MobileViT V1

MobileViT 是第一个基于轻量级 CNN 性能的轻量级 ViT 工作，性能 SOTA，优于 MobileNet V3、CrossViT 等。

MobileViT 块：将标准卷积的"展开+局部处理+折叠"中的局部建模替换为 Transformer 的全局建模，使模型兼具 CNN 的局部性和 ViT 的全局性。

图15.3.1：n×n卷积 → 1×1卷积(通道调整) → Unfold → Transformer(全局特征建模) → Fold → 1×1卷积(还原通道) → 残差拼接 → n×n卷积(特征融合)。

多尺度采样训练：在每个 GPU 迭代中随机抽样不同的空间分辨率，最大分辨率用最小批量加速优化，无需手动调整且提高了训练速度。

图15.3.2：标准采样固定分辨率批量训练 vs 多尺度采样随机分辨率动态批量训练。

15.3.2 MobileFormer 系列

MobileFormer：MobileNet + Transformer 并行

MobileFormer 通过双线桥将 MobileNet 和 Transformer 并行化，融合局部和全局表达能力。与 ViT 不同，它仅使用 6个或更少的可学习 tokens 作为全局先验，计算量更小。

图15.3.3：Mobile（MobileNet）处理图像提取局部特征，Former（Transformer）使用少量tokens学习全局先验，通过双向交叉注意力连接。

低成本双线桥：Mobile→Former 和 Mobile←Former 两种轻量级交叉注意力。在通道数较少的 Mobile 瓶颈处计算，将键（K）和值（V）的映射矩阵从 Mobile 中移除以节省计算，仅保留在 Former 中。

图15.3.4：每个MobileFormer块包含Mobile子块、Former子块、Mobile←Former（全局→局部）和Mobile→Former（局部→全局）四个部分。

  • Mobile 子块：使用动态 ReLU 替代 ReLU，参数由 Former 的第一个全局 token 通过两个 MLP 层生成
  • Former 子块：标准的 Transformer block，处理全局 tokens

15.3.3 EfficientFormer 系列

EfficientFormer V1

EfficientFormer 从时延分析出发，先找出 ViT 模型在端侧低效的根源，然后引入维度一致的 Transformer Block 作为设计范式，最后通过架构搜索获得不同系列模型。

尺寸一致性设计：将网络分割为两部分：

  • 4D 分区（MB[4D]）：操作符按 CNN 风格实现，使用池化混合器提取低级特征
  • 3D 分区（MB[3D]）：线性投影和注意力在 3D 张量上执行，发挥 MHSA 的全局建模能力

网络从 4D 分区开始，3D 分区应用于最后阶段。EfficientFormer 是一个完全基于 Transformer 的模型，无需集成 MobileNet 结构。

图15.3.5：PatchEmbed + 4个Stage（由MetaBlock组成）。前3个Stage为4D分区（CNN风格），最后Stage为3D分区（ViT风格）。

Transformer小型化小结

系列 核心思路 关键技术
MobileViT CNN+Transformer融合 局部卷积 + 全局Transformer建模、多尺度采样
MobileFormer MobileNet与Transformer并行 双向交叉注意力桥、少量全局tokens（≤6）
EfficientFormer 纯Transformer + 维度一致性设计 4D/3D分区、架构搜索、时延分析驱动

小结与思考

第15章覆盖了从 CNN 到 Transformer 的模型小型化全景。让我们用几个关键问题来收尾：

  • 为什么需要模型小型化？ 深度神经网络参数量庞大，面临内存不足和实时性不足的双重挑战，无法直接部署到移动端和嵌入式设备。模型小型化通过重新设计网络结构，在不严重影响精度的前提下降低参数量和计算复杂度。

  • CNN轻量化有哪些核心技术路线？ 主要有两条：人工设计（SqueezeNet的Fire Module、ShuffleNet的Channel Shuffle、MobileNet的深度可分离卷积、ESPNet的空洞卷积金字塔、GhostNet的廉价操作）和 NAS自动搜索（MobileNet V3、EfficientNet系列）。两条路线相互补充，共同推进了轻量化技术的发展。

  • 为什么连Transformer也需要轻量化？ Vision Transformer虽然效果好，但参数量动辄数十亿甚至数百亿。MobileViT、MobileFormer、EfficientFormer等工作将CNN的局部高效性与Transformer的全局建模能力结合，或通过维度一致性设计和架构搜索，使Transformer也能在移动端运行。

  • “小型化"和"压缩"是什么关系？ 本章讨论的"模型小型化"侧重于从结构设计入手构建轻量模型，而第16章的"模型压缩”（量化、剪枝、蒸馏、低秩分解）侧重于对已有模型进行后处理压缩。两者目标一致，但技术路径不同，实际部署中往往结合使用。

如果说第15章告诉我们"如何设计一个天生就小的模型"，那么即将到来的第16章将告诉我们"如何把一个已经很大的模型变小"——敬请期待模型轻量化四件套：量化、剪枝、知识蒸馏与低秩分解。

下一篇预告：第16章将深入模型轻量化技术，详解模型压缩四件套——量化（Quantization）、剪枝（Pruning）、知识蒸馏（Knowledge Distillation）与低秩分解（Low-Rank Factorization）。
