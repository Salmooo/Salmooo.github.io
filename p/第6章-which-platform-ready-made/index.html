第6章: Which Platform? — Ready Made

Jun 05, 2026
5 minute read

章节概览

第5章讲了"造"；第6章讲"买"——如何评估和选择商业现成的 FPGA 原型平台。

本章的核心论点是：互联灵活性（而非原始飞行时间）才是决定平台真正性能的关键因素。作者通过严谨的量化时序分析证明，即使是延迟互联（4ns 飞行时间）也能达到 100MHz+ 的系统性能，但互联资源的可用性——即能否提供足够的引脚来避免信号复用——才是实际的性能瓶颈。一个能让你以直连方式跑关键路径的平台，比一个飞行时间更短但逼你用 8:1 复用的平台更快。

本章最终给出了一份 Top-10 检查清单（Table 14），可作为任何商业 FPGA 原型板采购评估的标准化模板。

核心知识点

  • 商业原型板 ≠ 通用 FPGA 评估板：市场上很多"原型板"实际上是评估板、FMC 标准板或低成本目标板——它们不是本书讨论的 SoC 原型验证平台
  • 不同使用场景对板子特性的需求完全不同：RTL 调试需要极好的可见性和速度；软件验证几乎不需要调试功能但要求极高的鲁棒性和可复制性；评估板的分发要求低成本和高可靠性
  • 互联灵活性比飞行时间更重要：FPGA 内部路径延迟（~4.5ns）主导总路径延迟。直接互联飞行时间 1ns（总路径 5.5ns, ~180MHz）；延迟/交换互联飞行时间 4ns（总路径 9.5ns, ~105MHz）——差距仅约 2 倍，但灵活性带来的更好的分区方案往往远超这个差异
  • HapsTrak II（延迟互联）：FPGA 引脚全部引出到连接器→通过夹层连接板或排线在板间连接→每项目定制互联拓扑→项目结束后拆掉重新配置。连接器的粒度决定灵活性
  • CHIPit（交换互联）：可编程开关矩阵→以 8 路为粒度分配互联→可由分区工具实时控制→无需物理改动即可重新配置→支持远程操作
  • 互联可用性 > 飞行时间：如果分区需要 200 条路径而只有 100 个空闲引脚，2:1 复用 → 实际性能减半。灵活互联提供不同分区方案 → 释放更多引脚 → 消除复用需求 → 净性能可能更高
  • LVDS 高速 TDM 可达 1GHz：使用差分信令和源同步路径 → 8:1 复用 @800MHz 轻松支持 128MHz 内部逻辑 → 甚至 64:2 @30MHz+
  • HSTDM（高速时分复用）：HAPS-60 + Certify 工具自动化实现——板级 LVDS 能力 + 分区工具内置知识 = 自动化高性能复用
  • 永远无视供应商的门数宣称：跑自己的综合得到真实资源需求 → 加倍作为安全余量 → 与候选板的实际资源对比
  • 总拥有成本（TCO）≠ 购买价格：便宜板在数亿规模的 SoC 项目上出的问题，代价远超省下的钱

详细笔记

6.1 你需要板子做什么？

商业 FPGA 板市场鱼龙混杂——不同的板子为完全不同的目的设计和优化：

  • FMC 标准板（VITA 57）：为低量产物项设计，容量不够原型验证
  • 低成本目标板：为评估特定 FPGA 器件或 IP 而设计
  • 真正的 FPGA 原型平台：本书描述的对象

原型板的十一种使用目标：

  1. 验证原始 RTL 代码的功能性
  2. 测试开发中算法的系统内性能
  3. 验证外部 IP 的系统内操作
  4. 项目后期 ECO 的回归测试
  5. 物理层软件和驱动的系统内测试
  6. 应用软件与嵌入式 OS 的早期集成
  7. 为软件验证创建独立目标平台
  8. 为潜在合作伙伴提供评估或软件开发平台
  9. 建立公司级跨项目标准验证基础设施
  10. 未实现产品的展会展演
  11. 标准组织"PlugFest"兼容性测试

关键洞察：同一块板不可能同时最优地满足所有这些目标。例如——大量测试点的复杂调试板，不适合作为独立软件目标平台分发给软件团队。

外设模块化的逻辑相同：不要在 baseboard 上固定放置任何特定外设（USB、以太网等）。原因：

  • 不用的外设浪费 FPGA 引脚（最宝贵的资源）
  • 模块化子卡允许精确按需配置（包括 0 个）
  • IP 核通常只能通过模块化子卡支持（PHY 敏感、RTL 不可得）

HAPS 生态的例子：Synopsys 同时提供板和 IP → 同一供应商预测试过的 IP 子卡 → IP 升级到下一代标准时只需换子卡，不用替换整个平台。

“一般而言，终端用户应避免’一刀切’式的 FPGA 原型供应商选择方式，因为在大多数情况下，这会导致设计、项目和商业上的妥协。”

6.4 灵活性：互联

三种固定互联拓扑（图59）：

| 拓

扑 优势 劣势
菊花链（Daisy-Chain） 单一宽数据通路效率高 信号需穿越 FPGA（大量引脚浪费 + 路径延迟暴增）
星形（Star） 任意 FPGA 直接驱动任意 FPGA IO 引脚效率最低——所有 FPGA 的引脚硬连接到所有总线
混合 多数商业板的选择 固定的折中仍然不可能完美匹配每个设计

星形互联的经典陷阱（图60，延续第5章的例子）：

  • 3 个模块在 AHB 总线上 → 分区到 3 颗 FPGA
  • 第 4 颗 FPGA 的 130+ 引脚硬连接到总线 → 只能设高阻态 → 永久浪费
  • 如果用第 4 颗 FPGA 放设计的其他部分 → 引脚浪费可能致命

自建板 vs 商业板的关键差异：自建板可让互联精确匹配 SoC 顶层块图（尤其配合 DFP 方法时）。商业板有固定互联 → 必须妥协。

6.5 什么是最理想的互联拓扑？

答案：间接互联（Indirect Interconnect） — 两种形式。

形式一：延迟互联（Deferred Interconnect）= HapsTrak II

原理：FPGA 引脚不直接互连，而是全部引出到连接器 → 通过夹层连接板或柔性排线按需连接。

1

FPGA1 引脚 → 连接器 A ──(夹层板/排线)── 连接器 D → FPGA4 引脚

关键特性：

  • 可创建本地和远程连接
  • 项目结束后拆掉线缆，下一个项目重新配置拓扑
  • 连接器的粒度决定灵活性——粒度越细（每个连接器承载的引脚数越少），重组越灵活

形式二：交换互联（Switched Interconnect）= CHIPit

原理：可编程开关矩阵连接不同互联段。

粒度：CHIPit 系统以 8 路为单位 分配互联。

核心优势：

  • 可编程 → 无需物理改动即可重新配置 → 支持远程操作（远程软件验证实验室加载设计映像+配置交换机，完全不需要碰硬件）
  • 由分区工具控制 → 分区器发现需要更多连接时立即行使该选项 → 边分区边调整互联资源
  • 空闲引脚可在设计加载后重新路由到调试端口
  • 支持细粒度开关 → 解决分区问题用的最少必要连接数 → 其余本地连接仍可用于其他地方

现实限制：理论上理想的"任意点到任意点"全交叉矩阵难以扩展到大量连接。实际实现中会级联较小的开关矩阵并混合部分直接互联。

CHIPit 系统架构（图63）：

  • 同板 FPGA 之间有固定走线
  • 可编程开关提供额外连接路径
  • 类似开关可以链接存储模块、连接其他板/平台
  • 非所有开关组合合法且数量过多无法手工控制 → 由软件自动配置

即使直接互联，总延迟的主导因素也不是飞行时间而是 FPGA 内部路径。最大性能可达 250MHz（如果关键路径能分到直接互联上的话）。

交换互联更详细的分析：

  • 单个开关元件延迟：0.15ns ~ 0.25ns
  • 加上往返开关的板级走线：单次飞行时间约 1.5ns（最佳情况）
  • 实际设计中信号可能穿越多段（平均 2 级，极端 8 级，大设计分到 20 颗 FPGA 时）
  • → 需供应商提供路由优化工具 + 分区器协同控制路由器 → 最优结果

6.6.1 飞行时间到底有多重要？

核心洞察：互联可用性 > 飞行时间。

在大多数 FPGA 原型项目中，性能确实受限于互联——但限制因素是其可用数量而非飞行时间。

一个典型的例子：

  • 分区器要求两颗 FPGA 间 200 条路径
  • 实际只有 100 个空闲引脚
  • 无论是直接还是"从火星绕一圈"，都只能装 100 条
  • → 必须 2:1 复用 → 实际系统性能减半（内部逻辑最多以互联速度的一半运行）

灵活互联的价值：

  • 提供不同的分区方案 → 可能释放 100+ 额外引脚
  • → 200 条信号无需复用全速运行
  • → 灵活性产生的净性能可能远超直接互联的飞行时间优势

在速度与灵活性的权衡中，最优决策往往偏向灵活性。

总结：一个好的 FPGA 平台应同时提供多种选项——关键信号走直接互联，非关键信号走间接互联。识别非关键信号后甚至可以将其复用到共享路径上，为更关键的信号释放引脚。

6.7 速度：设计与布局的质量

高性能 PCB 的要求：

  • PCIe / DDR3 等接口原生运行需 100MHz+ → FPGA 的快速引脚 + PCB 的鲁棒设计
  • 最新高引脚数 FPGA 需要极复杂板设计——极少供应商能设计和制造 40 层板
  • 走线长度和阻抗匹配 → 良好差分信令 + 远程同步点间延迟匹配
  • 多批次一致性：不同批次甚至同批次不同板子间性能差异显著 → 设计质量差的标志

时钟和复位网络尤为重要：

  • 必须足够灵活容纳各种时钟源和频率
  • 必须在分布式时钟网络的每一点都交付优质时钟信号

电源同样关键：低阻抗、高电流路径到 FPGA 核心和 IO 电压轨引脚 → 维持低噪声（尤其每个时钟周期大量信号在 FPGA 间翻转时）。

两颗使用相同 FPGA 的板子可能看起来速度和质量差不多，但是板供应商的专业能力——而非 FPGA 芯片本身——决定它们能否可靠地交付原始性能。

6.8 板上信号复用支持

TDM 的基本概念：2+ 路信号共享同一条互联路径 → FPGA 内部需要 mux/demux 逻辑 + 两端同步。

Certify 的 CPM（Certify Pin Multiplier）：自动插入 TDM 逻辑、分析时序、选择哪些信号放入 MUX。

复用比与性能的跷跷板：

  • 复用比 8:1 + 内部逻辑跑 16MHz
  • → 要么降到 2MHz（几乎等于仿真器了）
  • → 要么外部信号以 128MHz+ 传输
  • → 需要板子支持高速外部信号 + 低噪声（否则毛刺破坏 TDM 流同步）

LVDS 高速 TDM（图65）

使用

FPGA 的 LVDS 差分信令能力 → 板上路径速度可达 1GHz。

HSTDM（高速 TDM）示例：

  • 800MHz 传输 + 8:1 复用 → 轻松支撑 128MHz 内部逻辑
  • 甚至 40:1 复用比也支持（只要板子质量够好）
  • 差分对 → 64:2 的 TDM 比率 → 30MHz+ 的原型 + 极其充裕的有效互联

HAPS-60 + Certify 的自动化 HSTDM：

  • 板级 LVDS 能力信息内置于分区工具
  • 工具自动分配差分对到最合适的信号
  • → 板和工具协同开发的价值

6.9 成本与鲁棒性

总拥有成本（TCO）≠ 购买价。需权衡：

  • 初始资本成本 vs 跨项目复用能力
  • 板子能否达到要求性能的风险
  • “为多年期数亿美元的 SoC 项目冒险使用一块便宜板子可能不是明智的”

鲁棒性特别重要：

  • 多数原型平台需要多份复制 → 分发给远程终端用户
  • 有些项目需要原型部署到真实环境（车载/客户/现场）
  • 板子必须可靠 + 鲁棒到能承受运输

“具有经过验证的异地部署和可靠性记录的板子，投资回收周期远短于需要持续关注才能保持运行的板子。”

供应商保修：应承诺保修期内立即从库存更换。

6.9.1 FPGA 供应决定板子交付

最新一代 FPGA 在早期产能爬坡期通常短缺 → 板供应商能否保障 FPGA 器件是关键的商业差异。板供应商必须是 FPGA 供应商的大批量客户才能确保自己的器件可获得性。

6.10 容量

永远不要轻信供应商的门数宣称。

LX550T vs LX760 的实际对比（表13）：

资源 XC6VLX550T XC6VLX760 比值
Logic Cells 549,888 758,784 138%
FF 687,360 948,480 138%
BlockRAM (Kbit) 22,752 25,920 114%
DSP 864 864 100%（相同！）
GTX 36 0 不适用

关键教训：资源对比不是线性的——不同设计中关键资源可能完全不同。DSP 相同的两块板，在算术密集型设计中实际上提供相同的有效容量。 而 LX760 没有 GTX——如果需要高速串行接口，LX550T 反而是更好的选择。

“在项目开始阶段节约板子容量成本，会在项目后期设计增长超出可用空间时浪费大量时间——这是很多人难以接受的建议。”

个人总结

这一章和上一章一起构成了"Make vs Buy"的完整论证。第5章教你怎么造，第6章教你如何买——但更重要的是，第6章悄悄论证了一个事实：对于大多数团队而言，“买"优于"造”。

作者的论证策略非常巧妙。他不是直接比较自建和购买的成本——那在第5章已经用"大多数自建板从未被复用"的事实暗示过。第6章专注论证的是一个更微妙但更重要的点：互联灵活性才是性能的真正决定因素，而非原始飞行时间。

这个结论的推论是：商业平台在互联灵活性上的投资（HapsTrak II 的延迟互联、CHIPit 的交换互联、HSTDM、由分区工具自动控制的路由）是单个自建项目难以复制的。这些不是硬件层面的创新——它们是硬件+软件+工具链的协同设计。一家公司可以设计一块板子，但几乎不可能为一两次自建项目开发出与商业平台同等水平的工具链支持。

最后，Top-10 检查清单的价值不仅在选板——它也反向定义了什么是一个好的原型验证平台。在后续章节讨论具体设计方法时，这十个维度可以作为判断"这对我选用的平台是否适用"的快速评估框架。

实践思考

  • Table 11 的价值在于迫使你承认：没有一块板同时擅长一切。精确理解你的主要使用场景，买最匹配的板，而不是最全的板
  • 互联可用性 > 飞行时间是一个反直觉但极其实用的工程准则。在选择板子时，问"能给我多少条有效路径避免复用"比问"最快能跑多少 MHz"更有实际意义
  • LVDS/HSTDM 是平台质量的放大器——一块板如果不能支持高质量差分信令，它的有效互联比标称值大打折扣
  • “门数"是一个有毒的营销指标。无视供应商的一切门数宣称；跑自己的综合、加倍结果、对比实际资源
  • 保修和交付可靠性应该被写入采购合同——如果你依赖原型平台来验证数亿规模的 SoC 项目，等 8 周才能拿到替换板是不可接受的
  • HAPS 和 CHIPit 代表了两种不同的哲学：HAPS 用物理上可重新配置的连接（延迟互联），CHIPit 用电子的可编程开关（交换互联）。前者在项目变化时可能需要动手，但路径质量更好；后者可以在几分钟内重新配置，甚至支持远程操作——选择哪种取决于团队的使用模式
