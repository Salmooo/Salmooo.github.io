AI系统：第10章 前端优化

《AI系统：原理与架构》学习笔记 — 深入AI编译器前端优化，涵盖计算图IR、算子融合、布局转换、内存分配、常量折叠、CSE、死代码消除、代数简化等核心技术

4 minute read

本章概览

如果说前面几章聚焦于 AI 硬件（GPU、TPU、NPU），那么第 10 章正式进入 AI 编译器的核心地带 —— 前端优化（Front-end Optimization）。前端优化发生在"开发者写的 Python 模型代码"和"硬件真正执行"之间的关键环节，它的使命是将高层计算图转换为更高效、更低开销的形式。

本章覆盖了 AI 编译器中 8 大前端优化技术：

  1. 图算 IR（计算图）：一切优化的基础 —— 计算图是什么、怎么生成、静态 vs 动态
  2. 算子融合：将多个算子"焊"在一起，减少访存和 Kernel 调度开销
  3. 布局转换：NCHW vs NHWC，数据在内存中怎么排最有效率
  4. 内存分配：静态内存 vs 动态内存，内存复用/替代/共享
  5. 常量折叠：编译期"提前算好"，省去运行时开销
  6. 公共子表达式消除（CSE）：重复计算只算一次
  7. 死代码消除（DCE）：删掉永远不会跑的代码
  8. 代数简化：利用交换律/结合律/分配律，数学上等价但更快

这 8 项技术层层递进，共同构成了 AI 编译器前端的"工具箱"。理解它们的原理，你就能真正看懂 PyTorch torch.compile、TensorFlow XLA、TVM 等编译器在背后做了什么。

10.2 图算 IR —— 一切优化的基石

10.2.1 计算图基本构成

计算图（Computational Graph） 是一个有向无环图（DAG），用于表示神经网络模型在训练与推理过程中的计算逻辑与状态。它由两个基本元素构成：

  • 张量（Tensor）：多维数据，具有形状（Shape）和元素类型等属性，在图中用有向边表示
  • 算子（Operator）：基本运算单元，在图中用节点表示，底层硬件实现称为 Kernel

节点间的有向线段既表示张量的流动，也描述了计算间的依赖关系。前向传播计算图 + 反向传播计算图合在一起，就构成了完整训练过程的全貌。

10.2.2 AI 框架如何生成计算图

自动微分的关键：开发者只需编写前向传播代码，AI 框架会自动追踪数据流，为每个参数精确微分，构建出完整的反向传播计算图。

两种生成方式：

模式 静态图（Eager 关闭） 动态图（Define-by-Run）
执行方式 先构建完整图，再执行 边定义边执行
即时结果 否 是
调试难度 难 简单
控制流 框架特定语法（如 tf.cond） 前端语言原生语法
性能 图优化空间大、性能更佳 优化受限、性能较差
内存 相对较少 相对较多
部署 可直接部署（含计算图信息） 不可直接部署

PyTorch 2.0 引入的 Dynamo 核心使命就是把动态图"转"成静态图，从而获得编译优化红利。

动态图转静态图的两种方式：

  • 基于追踪（Tracing）：以动态图模式执行并记录调度算子，保存为静态图模型
  • 基于源码转换（FX）：分析前端代码，自动将动态图代码转为静态图

10.2.3 计算图对 AI 编译器的作用

  1. 方便底层编译优化：计算图描述了全局信息，AI 编译器可以在执行前进行系统级优化 —— 分析数据依赖、做图优化 Pass、动态/静态内存优化、调整算子调度策略
  2. 分层优化便于扩展：将前端优化分为三个解耦层 —— 计算图优化 → 运行时调度优化 → 算子/Kernel 执行优化，各层独立 Pass，灵活扩展

10.3 算子融合 —— “少就是多”

10.3.1 为什么要做算子融合？

算子融合主要解决两大瓶颈：

  • 内存墙：访存瓶颈。将存在数据依赖的"生产者-消费者"算子融合，提升中间 Tensor 的访存局部性（Buffer 融合）
  • 并行墙：芯片多核增加与单算子多核并行度不匹配。将可并行的分支节点编排融合

10.3.2 四种经典融合方式

10.3.3 综合案例：复杂计算图"瘦身"

下图展示了一个更综合的融合案例：

通过横向融合 Enlarge + 纵向融合 + 激活融合，从复杂计算图得到一个极简版本 —— 减少了 Kernel 调度，实现了"少就是多"。

10.3.4 Conv-BN-ReLU 融合深度剖析

这是最经典的算子融合案例。先理解 BN 的计算流程：

BN 前向计算：求均值 \(\mu\) → 求方差 \(\sigma^2\) → 归一化 \(\hat{x}\) → 线性变换 \(y = \gamma \hat{x} + \beta\)

BN 反向计算：两次使用输入特征 \(X\) 及输出误差 \(\Delta Y\)，分别计算参数误差 \(\Delta\gamma, \Delta\beta\) 和输入误差 \(\Delta X\)。

融合的动机：Conv-BN-ReLU 三者在计算访存上存在大量中间结果的读写，融合后可大幅减少访存。

融合的核心思路是将 BN 拆成两个子层 BN_A 和 BN_B，分别融合到相邻的卷积层和激活层中，形成"归一化缩放 → 激活层 → 卷积层 → 计算卷积结果均值与方差"的流水线，整个过程只需一次读取和一次写回。

数学推导：将卷积公式代入 BN 公式 → 合并权重 \(w' = \frac{\gamma w}{\sigma}\) 和偏置 \(b' = \frac{\gamma(b-\mu)}{\sigma} + \beta\) → 再叠加上 ReLU → 得到融合后的单一算子 Conv2d_BN_ReLU。

10.4 布局转换 —— 数据怎么"摆"最省力

10.4.1 内存对齐：从硬件层面理解

处理器从内存读取数据是按**字（Word）**为单位（32 位系统 4 字节，64 位系统 8 字节），而非按字节。

没有内存对齐的代价：读取一个跨对齐边界的 INT（4 字节）需要两次总线操作 → 合并数据。

10.4.2 行优先 vs 列优先

二维张量（矩阵）在内存中的线性排布有两种基本方式：

核心原则：相邻操作访存地址应尽可能接近 —— 行优先遍历用行优先存储，列优先遍历用列优先存储（空间局部性原理）。

对于三维张量，按排列组合有 \(3! = 6\) 种排布方式（D1,2,3 / D1,3,2 / D2,1,3 / D2,3,1 / D3,1,2 / D3,2,1），选择相当复杂。

10.4.3 NCHW vs NHWC — GPU vs CPU 的分水岭

卷积神经网络的特征图通常用四维数组 [N, C, H, W] 存储。

NCHW（通道优先）：先取 W 方向 → 再取 H 方向 → 最后取 C 方向。同一通道的数值连续排布。

  • ✅ 适合 GPU：对大通道做 MaxPooling 等操作时，一次加载一整个通道，GPU 大内存带宽 + 强并行性完美匹配
  • ❌ 需要较大临时存储

NHWC（通道交错）：先取 C 方向（同一位置不同通道）→ 再取 W → 再取 H。

  • ✅ 适合 CPU：多核 CPU 可分别并行处理同一位置的不同通道，配合 1×1 卷积（Conv1×1）效率极高
  • ✅ 临时空间小，可边读边算
特性 NCHW NHWC
适合硬件 GPU 多核 CPU
通道数据 连续 交错
典型操作 MaxPooling、大通道卷积 Conv1×1、Depthwise Conv
TensorFlow 默认 channels_first channels_last

10.5 内存分配 —— 把有限的内存用到极致

10.5.1 为什么内存分配如此重要？

现代神经网络动辄数百亿参数。以 ImageNet 训练为例：

  • Inception v4（BatchSize=32）：需要约 40 GB NPU 内存
  • Wide ResNet-152（BatchSize=64）：需要约 180 GB NPU 内存

在 NPU 内存增长远慢于模型膨胀的现实下，高效的内存分配成为生死攸关的优化。

10.5.2 内存的划分：静态 vs 动态

静态内存（模型初始化时一次性申请，全程不变）：

  • 模型权重参数（Parameter）
  • 常量值节点（如归一化的均值/方差）
  • 模型输出

动态内存（随计算进行动态申请和释放）：

  • 输出张量（中间结果，用完即释放）
  • 工作区张量（临时缓冲区，如自定义 CUDA Kernel 的 workspace）

10.5.3 内存优化算法

四种策略：

策略 方法 典型应用
空间换内存 CPU Offload MoE 模型
计算换内存 梯度检查点（Gradient Checkpointing） 大模型训练
模型压缩 量化/剪枝/蒸馏 端侧推理
内存复用 分析数据流，重用内存 编译器优化

实际效果：MobileNet v2 经内存优化后，NPU 内存消耗明显减少，碎片数量显著降低。

两种核心复用操作

  1. 内存替代（Inplace Operation）：一块内存不再需要时，下一个 element-wise 操作原地覆盖。

  1. 内存共享（Memory Sharing）：两个数据大小相同且前一个不再需要时，后一个覆盖前一个。

复杂模型的内存优化过程

AI 编译器在初始化时感知每个算子的被使用次数并标记，然后按照生命周期进行 tag 分配、释放、复用的调度：

并行分配：不止考虑串行

内存分配还要考虑并行度。同样是有效的分配方案，右侧方案允许 \(A[2]\) 和 \(A[5]\) 并行执行，而左侧方案引入了不必要的依赖。

10.6 常量折叠 —— “能提前算的别拖到运行时”

10.6.1 传统编译器的常量折叠 

1

day_sec = 24 * 60 * 60  # 编译器直接替换为 day_sec = 86400

传统编译器通过常量传播（分析控制流图的可达性）识别常量表达式，在编译期求值替换。

10.6.2 AI 编译器的常量折叠

AI 编译器对计算图进行常量折叠，三种类型：

类型一：全常量输入节点

  • 如两个形状为 (N,C,H,W) 的四维常量 Tensor 做 AddN → 编译期直接算出结果替换节点

类型二：形状确定的 Shape 类操作

  • 如 Size、Shape 等操作只依赖输入形状而非具体值 → 形状在编译期确定即可折叠

类型三：链式折叠

  • 节点输入不全为常量，但经过一系列折叠后可变为常量节点

注意：如果常量太大（以字节为单位），AI 编译器可能选择不替换，防止计算图变得过大。

10.7 公共子表达式消除（CSE） —— 重复计算只算一次

传统编译器的 CSE 

1
2
3

temp = b * c
a = b * c + g    # → a = temp + g
d = b * c + e    # → d = temp + e
  • 局部 CSE：限于基本块内（如 LVN 算法）
  • 全局 CSE：跨多个基本块（如 LCM 算法）

AI 编译器的 CSE

AI 编译器在计算图中搜索相同结构的子图，将所有不同输出连接到同一个子图上，然后通过死代码消除删除重复子图。

TensorFlow 的实现思路：

  1. 反向 DFS 遍历获取逆后续节点集（确保处理某节点时其输入已处理完）
  2. 用混合哈希为每个操作节点计算签名（输出节点数、类型、输入节点信息等）
  3. 维护候选集，新节点哈希命中则复用已有节点，旧节点留待 DCE 删除

10.8 死代码消除（DCE） —— 删掉永远不会跑的代码

传统编译器的 DCE

不可达操作两类：

  • 不可达基本块中的操作（控制流图中不存在从入口基本块到达的路径）
  • 条件分支优化导致的舍弃分支中的操作

无用操作：结果无外部可见效应的操作（如定义后从未使用的变量赋值）。

AI 编译器的 DCE

在计算图中：

  • 不可达节点：从输入节点通过有向边无法到达的节点 → 直接删除
  • 无用节点：输出没有后继节点，不影响最终输出 → 直接删除

训练时产生的大量子图在推理时通常是无用的，DCE 可以大幅压缩推理模型。

10.9 代数简化 —— 用数学"变形"加速计算

10.9.1 算术简化：结合律、交换律、分配律

结合律简化：

\((A \star B) \otimes ((A \star B) \otimes C) \rightarrow (A \star B) \otimes C\)

其中 \(\star\) 为 Conv，\(\otimes\) 为 MatMul。

交换律简化：

\(\text{ReduceSum}(\text{BitShift}(A)) \rightarrow \text{BitShift}(\text{ReduceSum}(A))\)

先 ReduceSum 降维，再 BitShift，运算开销大幅减少。

分配律简化（提取公因式）：

\((A \cdot B) \star C + (A \cdot B) \star D \rightarrow (A \cdot B) \star (C + D)\)

将 4 次算子操作降为 3 次。

⚠️ 注意：矩阵乘法不满足交换律（\(AB \neq BA\)）！应用代数简化前必须先确认算子是否满足对应的运算律。

10.9.2 运行简化：消除冗余操作

对合算子简化（逆函数 = 自身函数，两次操作等于无操作）：

\(\text{Op}(\text{Op}(x)) \rightarrow x\)，如取反、倒数、逻辑非、矩阵转置等。

幂等算子简化（多次操作等价于一次）：

\(\text{Op}(\text{Op}(x)) \rightarrow \text{Op}(x)\)，如 \(\text{Reshape}(\text{Reshape}(x,\text{shape1}),\text{shape2}) \rightarrow \text{Reshape}(x,\text{shape2})\)

10.9.3 广播简化：先算小的再广播

\((S1 + \text{Mat1}) + (S2 + \text{Mat2}) \rightarrow (S1 + S2) + (\text{Mat1} + \text{Mat2})\)

两个向量先相加（维度小）再广播到矩阵维度，比先各自广播再相加节省一次广播开销。

    ├── 布局转换（优化数据访存模式）
    ├── 内存分配（高效复用有限 NPU 内存）

    └── 代数简化（数学等价变换）

核心启示：

  1. 一切优化始于计算图：AI 编译器先拿到全局计算图，才能做各种图级别优化 Pass，就像 LLVM 基于 IR 做优化一样
  2. 访存是最大的瓶颈：算子融合、布局转换、内存分配——三项技术的底层目标都是减少对主存的访问、提升片上缓存利用率（对抗"内存墙"）
  3. 编译期能做的事别拖到运行时：常量折叠、CSE、DCE、代数简化——在编译期就"算好"、“去重”、“删掉”
  4. 数学是优化的好朋友：代数简化利用结合律/交换律/分配律，在不改变计算结果的前提下调整执行顺序，实现"数学上等价、性能上更优"
