AI系统：第2章 AI计算体系

《AI系统：原理与架构》学习笔记 — 深入理解AI芯片、计算模式与硬件设计的底层逻辑

3 minute read

本章概览

如果说第1章是站在山顶俯瞰AI系统的全景地图，那第2章就是拿上放大镜，一头扎进AI计算的"地基"——芯片和计算体系。这一章会带你搞清楚几个"硬核"问题：

  • AI芯片到底是什么？CPU、GPU、FPGA、ASIC 它们各自在AI这场大戏里扮演什么角色？
  • 摩尔定律是不是真的"凉了"？后摩尔时代芯片架构该怎么玩？
  • 神经网络里那些矩阵乘、卷积，到了芯片底层到底是怎么算的？
  • 为什么模型要量化、要剪枝、要轻量化？这对芯片设计意味着什么？
  • 如何用一张 Roofline 图，一眼看穿你的模型在特定芯片上是"算不动"还是"喂不饱"？

别被这些术语吓到，咱们慢慢聊，你会发现硬件其实也没那么"硬"。

2.1 引言：AI芯片的前世今生

2.1.1 芯片 + AI = ？

先来一个灵魂拷问：什么是芯片？

说白了，芯片就是在半导体衬底上"雕刻"出的一堆电路，能让它实现特定功能。在半导体被发明之前，人类只能"用机械控制电"——笨重、低效；而半导体出现后，我们可以直接"用电来控制电"。芯片，这个小小的物理接口，一手托着物质世界，一手牵着数字世界，说它是两个世界之间的"唯一翻译官"也不为过。

那把"AI"和"芯片"放在一起呢？

AI的发展轨迹很有意思：早期因为算力不够、数据不够、算法也不够，AI一直处于"叫好不叫座"的状态。那时候大家图省事，拿现成的通用CPU跑AI任务——虽然性能不咋地，但胜在不用额外砸钱研发专用芯片。可随着深度学习越来越成熟、ChatGPT这种"核弹级"应用横空出世，通用芯片的性能瓶颈一下子就暴露了。于是，专为AI设计的ASIC芯片就成了兵家必争之地。

简单来说：AI越火，算力缺口越大 → 算力缺口越大，专用芯片需求越猛 → 芯片越猛，AI又能跑得更远。这是一个正向飞轮。

2.1.2 四类芯片，四种打法

按技术架构，AI芯片圈主要有四大流派，各有各的绝活：

芯片类型 一句话描述 优势 劣势
CPU 计算机的"瑞士军刀" 逻辑控制强，什么AI模型都能跑 并行计算弱，效率不高
GPU “暴力计算"专家 几千个核心并行干活，矩阵乘算得飞快 功耗高，得CPU"伺候"着才能工作
FPGA 硬件界的"变形金刚” 可重配置，灵活性极高，整数运算强 编程门槛高，内存带宽不如GPU
ASIC 为AI"量身定制" 性能最强、功耗最低 研发周期长，烧钱，改不了

CPU 走的是冯·诺依曼架构的老路子——取指、解码、执行、访存、写回，一步步来。它有大把缓存和复杂的控制逻辑，适合搞逻辑控制、串行计算，但面对AI这种"暴力并行"的需求就有点力不从心了。

GPU 则是另一个极端。它砍掉了大量数据预取和决策模块，把宝贵的芯片面积都给了计算单元（ALU），所以并行计算效率甩CPU好几条街。但GPU有个"公主病"——必须由CPU调度才能工作，而且功耗不小。后来人们把GPU的能力扩展到通用计算领域，就有了GPGPU（通用GPU）。

FPGA 的思路最"野"：芯片里集成了海量基本门电路和存储器，用户通过更新配置文件来定义它们怎么连接。这意味着你可以针对不同算法做不同硬件设计！常用于深度学习推理阶段。国外的Xilinx、Altera是老大，国内复旦微电子、紫光同创也在追赶。

ASIC 是终极方案——需求定制、专门设计、特定强化。NPU（神经网络处理器）、TPU（张量处理器）都属于这一类。因为是为AI量身打造的，性能往往碾压GPU和CPU。现在国内外大多数AI芯片创业公司都在走这条路。

2.1.3 摩尔定律：一个持续了半个世纪的"预言"

1965年，英特尔创始人戈登·摩尔放了个"预言"：集成电路上的元器件数量，每隔18到24个月就会翻一倍，性能也跟着翻倍。 换句话说，你花同样的钱，每两年就能买到性能翻倍的电脑。

这个"预言"居然奇迹般地兑现了半个多世纪，成了半导体行业的"黄金准则"。

但到了2017年，图灵奖得主 David Patterson 公开宣告："摩尔定律真的结束了，但计算机体系结构将迎来下一个黄金时代。" 现在我们已经进入后摩尔时代——虽然制程工艺还在往前拱，但翻倍的节奏已经越来越慢了。

从芯片架构的演进来看（图2.1.3），这条路走得相当精彩：

1

单核CPU → 多核CPU → 多核GPU/NPU → 超异构集群体系

超异构集群是目前的终极形态——把CPU、GPU、NPU等各种处理器通过高速网络捏在一起，根据任务需求动态调配资源。这就像一支乐团，不同乐器各司其职，指挥（调度系统）负责让它们和谐演奏。

正如书中所说：“加油吧，下一个黄金时代的从业者们！” 这句话送给所有在AI芯片领域摸爬滚打的人。

2.2 AI 计算模式：算法怎样"塑造"芯片

搞清楚了芯片有哪些类型，接下来我们看看AI算法对芯片设计提出了哪些"奇葩"需求。毕竟，不理解上层计算负载，就没法做好底层硬件设计。

2.2.1 经典模型结构：神经网络长什么样

神经网络这个东西，说白了就是模仿人脑搞出来的一堆"人工神经元"。它们分层排列，每个神经元和下一层的所有神经元都连着，每条连线上有个可调节的权重。

一个神经网络模型由这些基本组件构成：

  • 神经元（Neuron）：基本计算单元，说白了就是模型里的权重参数
  • 激活函数（Activation）：给输出加"非线性"调料——没有它，无论堆多少层，本质上还是个线性变换。常见的有 Sigmoid、ReLU、Tanh
  • 前向传播：数据从输入层一路"流"到输出层
  • 反向传播：从输出端往回算梯度，告诉每个权重"你该往哪个方向调整"
  • 损失函数（Loss）：衡量"你预测的"和"真实的"差多远
  • 优化算法：梯度下降那一套，决定权重怎么更新

一个有趣的数字：神经网络中 90% 以上的计算量都是乘加操作（MACs），也就是"权重求和"。所以芯片设计的核心命题之一就是：如何高效做乘加。

神经网络的诞生分为两个阶段：训练和推理。训练阶段算力消耗大、耗时长，通常在云端服务器进行；推理阶段则要求响应快、资源省，可能在手机、摄像头等边缘设备上跑。所以很多芯片公司打的第一仗往往是"推理芯片"。

主流网络结构有以下几种：

  1. 全连接层（FC）：每个神经元和上一层所有神经元相连，密集得像个"交际花"。通常放网络最后几层做分类。
  2. 卷积层（Conv）：用可学习的卷积核在输入上"滑动扫描"，提取局部特征。参数共享是它的精髓——同一个卷积核扫遍整张图，参数量大减。
  3. 循环神经网络（RNN/LSTM/GRU）：专门处理序列数据，有"记忆"能力。但传统RNN有梯度消失的问题，LSTM用"门控机制"解决了这个痛点。
  4. 注意力机制（Attention）：2017年 Transformer 架构的核心组件。让模型学会"该看哪里"，彻底改变了NLP和CV的格局。现在的大模型（GPT、LLAMA）底层都是它。

从芯片设计的角度看：你得同时支持卷积/全连接这种计算密集型算子，也要支持 Softmax、LayerNorm 这种访存密集型算子。这就是为什么AI芯片的设计如此复杂——它不能偏科。

2.2.2 模型量化与剪枝：给模型"减肥"

训练完的大模型动不动就几百GB，要把它塞进手机或者嵌入式设备里，怎么办？两条路：

量化（Quantization）：把高精度数据（比如FP32）转成低精度（比如INT8）。就像把高清照片压成缩略图——画质差了点，但文件小了几十倍。

好处很实在：

  • 省内存：数据位宽降了，存储需求自然降了
  • 省算力：两个INT8相乘比两个FP32相乘，硬件开销小得多
  • 省电：数据搬移少了，功耗就降了——对于电池供电的设备，这就是命
  • 部署灵活：更容易在资源受限的设备上跑

剪枝（Pruning）：找出模型中"不重要"的权重，咔嚓一刀剪掉。就像修剪一棵树，去掉多余的枝叶，树形更紧凑。剪完后模型体积变小、计算量减少，精度却不会降太多。

从芯片设计角度思考：既然量化和剪枝是趋势，芯片就该：

  • 支持多种位宽的计算单元（FP8、INT8、INT4 都来一套）
  • 提供多种存储格式（FP16、BF16 按需选择）
  • 内置稀疏计算加速逻辑（英伟达A100就干了这事）
  • 加入数据压缩模块，减轻内存带宽压力

2.2.3 轻量化网络：用小模型撬动大世界

把神经网络跑在手机上，不能光靠量化和剪枝——网络本身的设计也得"瘦身"。这就催生了轻量化网络的设计方向。

两个关键指标：

  • 参数量（Params）：模型里有多少个可学习的权重
  • 计算量（FLOPs）：需要多少次浮点运算

轻量化的三大设计思路：

  1. 减少内存空间：用两个 3×3 卷积核替代一个 5×5 卷积核，参数量直接降到原来的 18/25，还能保持相同的感受野。
  2. 减少通道数：MobileNet 搞出了深度可分离卷积（Depthwise + Pointwise），一个 3×3×16×32 的标准卷积（4608个参数）可以被替换成 3×3×16 + 1×1×16×32（656个参数），少了整整7倍！
  3. 复用特征图：DenseNet 和 GhostNet 的思路——前面层算出来的特征图，后面层接着用，别浪费。

这些设计花样繁多，对芯片来说意味着：你得支持各种奇形怪状的卷积核尺寸、不同的卷积方式（Depthwise、分组卷积、通道混洗……），以及跳跃连接、密集连接这些花式连接方式。

2.2.4 分布式并行：大模型的"分而治之"

大模型动辄几千亿参数，单张卡根本装不下。于是有了分布式并行。

先了解几个通信"暗号"（集合通信原语）：

  • All-Reduce：所有人把数据汇总，算完再广播回去。训练时同步梯度就用它。
  • All-Gather：每个人的数据都广播给所有人。
  • Broadcast：一个人广播，其他人接收。常用于分发模型参数。
  • Reduce / Scatter / Gather：各种收集和分发花样。

并行策略分两大类：

数据并行：同一个模型复制到多张卡上，每张卡吃不同的数据，各自算梯度，然后同步。

  • DP（最简单）→ DDP（分布式版本）→ FSDP（连优化器状态都分片，极致省内存）

模型并行：

  • 张量并行：把一层的大矩阵拆成几块，分到不同卡上同时算
  • 流水并行：不同层分给不同卡，像流水线一样接力处理

从芯片设计角度看：需要高带宽低延时通信、灵活的计算资源调度、以及端到端的优化（比如英伟达的 Transformer Engine 就是专为大模型定制的）。

2.3 关键设计指标：如何"打分"一款AI芯片

2.3.1 先把"行话"学会

看AI芯片发布会，总会遇到这些单位：

缩写 全称 含义
OPS Operations Per Second 每秒操作数，1 TOPS = 每秒一万亿次
FLOPS Floating Point Ops/Sec 每秒浮点操作数
MACs Multiply-Accumulate 乘加操作，1 MAC ≈ 2 FLOPs
FLOPs Floating Point Operations 浮点运算次数（衡量模型复杂度）
MAC Memory Access Cost 内存占用量（衡量模型内存开销）

2.3.2 六大关键指标

评价一款AI芯片，不能只看算力——得全方位考量：

  1. 精度（Accuracy）：包括"计算精度"（支持FP32还是FP16？）和"模型效果精度"（跑ImageNet准确率多少？）
  2. 吞吐量（Throughput）：单位时间处理多少数据，和核心数强相关
  3. 时延（Latency）：从输入到输出的等待时间——自动驾驶场景里，多1ms可能就是生死之差
  4. 能耗（Energy）：高性能通常意味着高功耗，移动设备和IoT场景必须精打细算
  5. 价格（Cost）：硬件自身成本 + 系统集成成本 + 软件生态成本，一笔都不能少算
  6. 易用性（Flexibility）：文档齐不齐？SDK好不好用？调试工具方不方便？这些"软实力"往往决定芯片能不能落地

2.3.3 两个核心优化方向

芯片设计的核心矛盾可以归结为两个字：MACs 和 PE。

优化 MACs：

  • 减少没用的 MACs → 增加稀疏计算硬件结构
  • 降低单次 MACs 执行时间 → 提频、减指令开销

优化 PE（处理单元）：

  • 增加 PE 数量 → 更先进的制程工艺，同样面积塞更多计算单元
  • 提高 PE 利用率 → 软硬件协同，让每个PE都"忙起来"，别闲着

2.3.4 Roofline Model：一张图看透性能瓶颈

有了芯片参数和模型参数，怎么快速判断瓶颈在哪？Roofline Model 是绝佳工具。

核心概念——算术强度（Arithmetic Intensity）：

\[ \text{算术强度} = \frac{\text{模型运算量 (ops)}}{\text{模型内存占用量 (bytes)}} \]

把这个值和芯片的操作字节比（算力÷带宽）对比：

  • 如果算术强度 低 → 带宽受限（数据搬得太慢，计算单元在"等米下锅"）
  • 如果算术强度 高 → 算力受限（计算单元忙不过来）

举个例子：V100 GPU 的 FP16 峰值算力 125 TFLOPS，片外带宽 900 GB/s。对于一个 (8192, 128, 8192) 的矩阵乘，算术强度 ≈ 124.1，低于操作字节比 138.9 → 带宽受限。但如果矩阵变成 (8192, 8192, 8192)，算术强度飙到 2730 → 算力受限。

Roofline 图上的每一步"爬升"都对应着一种优化手段——调软件策略、加PE数量、扩内存容量、提带宽……这张图就是芯片性能优化的"藏宝图"。

2.4 核心计算之矩阵乘：AI芯片的"心跳"

2.4.1 卷积 ≈ 矩阵乘

说出来你可能不信：卷积操作本质上就是一个矩阵乘法。

通过一个叫 Im2Col（图像转列）的操作，把输入特征图按照卷积窗口重新排列，卷积核也排成矩阵，然后——噔噔，卷积就变成了矩阵乘！

为什么要费劲做这个转换？因为矩阵乘是计算机科学里被研究得最透彻、优化得最极致的运算之一。把卷积变成矩阵乘，就能继承几十年的优化成果。

2.4.2 分块（Tiling）：大矩阵的"化整为零"

转换后的矩阵往往大得离谱，芯片里的内存根本放不下。怎么办？分块。

原理很简单：把大矩阵切成一个个小块，每次只加载一小块到片上缓存，算完一部分结果再加载下一块。这就像吃披萨——谁也不会整张往嘴里塞，都是一片一片来。

分块的妙处在于数据复用：一块数据加载到离计算单元最近的缓存后，可以被反复使用多次，极大减少了对外部内存的访问。这就是"用空间换时间"的经典玩法。

2.4.3 矩阵乘库与优化

CPU上有 OpenBLAS、Intel MKL，GPU上有 cuBLAS、cuDNN——这些都是专门为矩阵乘做了极致优化的库。它们的核心逻辑分两步：

  1. 识别输入矩阵的形状
  2. 从"武器库"里选最优的 Kernel 来执行

矩阵乘优化的招数总结：

优化方向 具体手段
循环优化 调循环顺序、改善缓存命中率
分块矩阵乘 大矩阵切小块，降低复杂度、利用缓存
SIMD指令 SSE/AVX，一条指令同时处理多个数据
SIMT多线程 多核并行，任务拆分
算法改进 Strassen算法、FFT变换等

硬件层面的终极目标就两条：

  • 让每条指令执行更多的 MACs（SIMD/Tensor Core）
  • 在同样带宽下，每个时钟周期执行更多 MACs（低比特计算，比如8bit MACs 比 32bit 多4倍）

2.5 计算之比特位宽：位宽越小，世界越大

2.5.1 数据在计算机里长什么样

整数用二进制补码表示，最高位是符号位（0正1负）。

浮点数遵循 IEEE 754 标准，由三部分组成：

  • 符号位（S）：1 bit，管正负
  • 指数位（E）：决定数值范围有多大
  • 尾数位（M）：决定小数点后有多精确

一个单精度浮点数（FP32）= 1位符号 + 8位指数 + 23位尾数 = 32位。而双精度（FP64）则扩展到 1+11+52=64位，精度更高，范围更大。

2.5.2 AI模型里的"数据类型全家桶"

来看看AI领域常用的数据类型：

类型 位宽 特点 适用场景
FP32 32bit 单精度，范围大 训练标配，精度有保障
TF32 19bit 英伟达搞的"缩水版FP32"，砍了13位尾数 A100开始支持，训练加速
FP16 16bit 半精度，范围较窄 混合精度训练，省显存
BF16 16bit 指数位和FP32一样宽（8位），动态范围大 大模型训练首选，Transformer友好
FP8 8bit H100新品，E4M3和E5M2两种口味 LLM领域探索中，极致的省
INT8 8bit 整数，简单粗暴 推理量化，嵌入式部署

BF16 是个有趣的设计：直接把 FP32 的前16位截下来用。指数位宽和 FP32 一样（都是8位），所以动态范围不变，但精度低了一点。这个"简单粗暴"的设计在 Transformer 大模型上表现意外地好。

2.5.3 为什么要降低位宽？一个数字就懂了

操作 能耗 (pJ) 面积 (μm²)
8bit 加法 0.03 36
32bit 加法 0.1 127
16bit FP 加法 0.4 1360
32bit FP 加法 0.9 4184
8bit 乘法 0.2 282
32bit 乘法 3.1 3495
32bit SRAM 读取 5 N/A
32bit DRAM 读取 640 N/A

看到没？32bit 乘法的能耗是 8bit 乘法的 15 倍，面积是 12 倍！而从 DRAM 读一次数据的能耗（640 pJ）更是比算一次 8bit 加法（0.03 pJ）高了 2 万倍。

这就是为什么"降低比特位宽"是AI芯片的永恒主题——数据搬移比计算贵得多。

小结

第2章从硬件底层出发，帮我们建立了一个完整的AI计算体系认知：

  • 芯片格局：CPU/GPU/FPGA/ASIC 各有各的主场，ASIC是未来AI芯片的主赛道。摩尔定律虽已放缓，但体系结构创新才刚刚开始。
  • 算法驱动硬件：神经网络的计算模式（卷积、Attention、量化、剪枝、分布式并行）直接决定了芯片该怎么设计。不理解上层负载，就做不好底层硬件。
  • 性能评估：用算术强度和 Roofline Model 可以快速判断模型在特定芯片上是"带宽受限"还是"算力受限"——这是性能优化的起点。
  • 矩阵乘：AI计算的灵魂操作。从卷积到矩阵乘的转换（Im2Col）、分块策略（Tiling）、多级缓存优化，这些构成了AI芯片性能的基石。
  • 比特位宽：越低越好（在精度允许的前提下）。从FP32到FP8，从训练到推理，位宽的每一次"瘦身"都意味着能耗和面积的巨大节省。

核心启示：AI 系统是一个"软硬件协同"的游戏——上层算法定义计算模式，底层芯片提供算力支撑，中间编译和框架层负责把两者高效地"翻译"到一起。理解这个链条的每一环，才能真正做好AI系统。
