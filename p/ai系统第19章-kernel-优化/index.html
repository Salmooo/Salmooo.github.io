AI系统：第19章 Kernel 优化

《AI系统：原理与架构》学习笔记 — 深入推理引擎 Kernel 层优化，涵盖卷积计算原理、Im2Col 算法、Winograd 算法、QNNPACK 间接卷积与推理内存布局

5 minute read

本章概览

如果说 Runtime 层是推理引擎的"总调度"，那么 Kernel 层就是推理引擎的"肌肉"——所有底层数学运算（矩阵乘、卷积、激活函数等）最终都在这里执行。Kernel 层的性能直接决定了推理的速度和效率。

本章从五个维度系统讲解 Kernel 优化技术栈：

  1. Kernel 层概述：硬件平台优化（NEON/AVX/CUDA/OpenCL/Vulkan）+ 封装算子库（cuDNN/MKLDNN）+ 推理流程与开发范式
  2. 卷积计算原理：数学定义 → 离散卷积 → CNN 卷积（Padding/Stride/Channel/Kernel）+ 时间复杂度/空间复杂度分析
  3. Im2Col 算法：卷积转矩阵乘的核心思路，空间换时间，配合 GEMM 库加速
  4. Winograd 算法：减少乘法次数（4 次乘法 replace 6 次），加减法换乘法，适用于小卷积核（3×3）
  5. QNNPACK 间接卷积：消除 Im2Col 内存拷贝，通过间接缓冲区 + PDOT 微内核优化移动端量化推理
  6. 推理内存布局：CPU/GPU 多级缓存架构，NCHWX 通道拆分与 SIMT 适配

一句话总结：Kernel 优化的本质是"用空间换时间，用加法换乘法，用局部性换缓存命中"。从 Im2Col → Winograd → QNNPACK，每一次演进都在更精细地平衡这三者。

19.1 Kernel 层概述

19.1.1 Kernel 层定义与定位

Kernel 层是推理引擎中执行底层数学运算的核心组件，负责将上层调度（Runtime）的指令翻译为具体硬件上的计算操作。推理引擎的整体架构中，Kernel 层处于最底层，直接与硬件打交道。

CPU 优化

技术 平台 特点
NEON ARM 架构 SIMD 扩展，移动端/嵌入式设备推理优化
AVX / AVX2 Intel x86 高级向量扩展指令集，浮点/整数运算加速
Metal Apple 低级图形+计算 API，优化 Apple GPU 上的性能

GPU 优化

技术 平台 特点
CUDA NVIDIA 并行计算平台，大规模矩阵运算和卷积
OpenCL 跨平台 异构系统开放标准，CPU/GPU 通用
Vulkan 跨平台 新一代图形+计算 API，高性能计算场景

封装的高性能算子库

  • cuDNN：NVIDIA 为 GPU 优化的深度神经网络算子库（卷积、池化、归一化、激活函数）
  • oneDNN（原 MKLDNN）：Intel 为 CPU 优化的深度神经网络算子库，支持多种 Intel 处理器

实际应用中，开发者需要根据目标设备和性能要求选择最合适的优化策略组合。

19.1.2 Kernel 层优化方法

Kernel 层优化从四个维度展开：

1. 算法优化

主要针对卷积算子的计算优化，核心算法包括：

  • Im2Col + GEMM：将卷积转化为矩阵乘法，利用高性能 BLAS 库加速
  • Winograd：通过数学变换减少乘法次数，用加法换乘法
  • 这些算法通过特定的数学变换降低计算复杂度，提升推理速度

2. 内存布局

  • 权重和输入数据重排：使数据访问更连续，减少缓存缺失
  • Channel First（NCHW）vs Channel Last（NHWC）：根据硬件选择通道存储顺序
  • NC1HWC0 / NCHW4：将 C 维度拆分（C1 分组数，C0 每组通道数），适配 SIMD 操作和 Winograd 算法，提高缓存利用率

3. 汇编优化

  • 指令优化：针对特定 CPU 指令集（AVX、SSE 等）进行向量化
  • 循环优化：减少循环开销，增加指令级并行性
  • 存储优化：优化内存访问模式，提高缓存命中率

4. 调度优化

  • 并行计算：根据操作间依赖关系合理调度任务
  • 自动调优：减少等待时间，提高资源利用率

19.2 卷积计算原理

19.2.1 卷积的数学原理

卷积是对两个实变函数的一种数学运算。在泛函分析中，通过函数 \(f\) 和 \(g\) 生成第三个函数的积分变换：

\[ (f * g)(x) = \int_{-\infty}^{\infty} f(\tau) g(x - \tau) d\tau \]

通俗理解：卷积包含"卷"和"积"两步：

  • “卷”：将函数 \(g\) 翻转并平移（\(x + y = t\) 约束）
  • “积”：对重叠部分做乘积并积分

卷积的核心性质：

  • 交换律：\(f * g = g * f\)
  • 结合律：\((f * g) * h = f * (g * h)\)
  • 分配律：\(f * (g + h) = f * g + f * h\)

物理意义：在不同领域有不同解释：

  • 信号处理：系统对输入信号的累积响应
  • 图像处理：滤波和特征提取（边缘检测等）
  • 概率论：两个独立随机变量之和的概率密度函数

离散卷积

\[ S(i, j) = (K * I)(i, j) = \sum_{m}\sum_{n} I(i+m, j+n) K(m, n) \]

离散卷积可以看作矩阵乘法——输入矩阵的每一行与上一行对应位置平移一个单位的元素相同，这种矩阵称为 特普利茨（Toeplitz）矩阵。

在 CNN 术语中，第一个参数叫 Input，第二个叫 Kernel Function，输出叫 Feature Map。这些多维数组统称为张量（Tensor）。

19.2.2 CNN 卷积计算

CNN 中的卷积通常指由多个并行卷积组成的计算——单个核只能提取一种特征，网络需要每层在多个位置提取多种类型的特征。

关键名词

名词 含义 要点
Padding 在输入周围添加行列（通常填 0） Valid（不填）/ Same（输出=输入尺寸）/ 自定义
Stride 卷积核每次滑动的距离 增大 stride 减小输出尺寸，加快计算，降低特征提取能力
Channel 特征图数量 / 深度 增加 channel 增强特征提取能力，但增加计算量
Kernel 可学习的卷积核参数 参数共享，在图像上滑动，每个 kernel 对应一张特征图

计算复杂度分析

时间复杂度（单卷积层）：

\[ O(f_{in}^2 \times C_{in} \times C_{out} \times K^2) \]

空间复杂度（访存量）：

\[ \text{总参数量} + \text{各层输出特征图大小} = \sum_{l=1}^{D} K_l^2 \cdot C_{l-1} \cdot C_l + \sum_{l=1}^{D} f_l^2 \cdot C_l \]

19.2.3 卷积优化手段

Tensor 运算

张量（Tensor）是机器学习的基础数据结构——0 维是标量、1 维是向量、2 维是矩阵、3 维是立方体。

张量计算 = 维度处理 + 数据处理，无副作用（函数式编程风格）。

Tensor 内存布局

格式 全称 默认框架 特点
NHWC 样本数-高-宽-通道 TensorFlow / 多数推理引擎 Channel 在最后
NCHW 样本数-通道-高-宽 PyTorch Channel 在 H/W 之前

不同内存布局影响内存访问的连续性，进而影响缓存效率——这也是后续 Im2Col 等算法要解决的核心问题。

19.3 Im2Col 算法

19.3.1 核心思想

Im2Col = Image to Column，将三维张量转换为二维矩阵，使卷积操作等价于矩阵乘法，从而利用高度优化的 GEMM 库（BLAS、MKL）加速。

核心原理：用空间换时间 —— 将原本可复用的数据平坦地分布到矩阵中，增加了约 (KH×KW−1) 倍的内存占用，但换来了连续内存访问和高效矩阵乘法。

每个窗口的卷积 = 两个一维向量的点乘。将所有窗口展开成行向量，卷积核展开成列向量，卷积 → 矩阵乘。

19.3.2 算法过程

Im2Col + MatMul 的四个步骤：

步骤一：Input 重排

将输入 \(N \times \text{IH} \times \text{IW} \times \text{IC}\) 展开为 \((\text{OH} \times \text{OW}) \times (N \times \text{KH} \times \text{KW} \times \text{IC})\) 二维矩阵。

每个行向量 = 一个卷积窗口覆盖的所有数据（先排第一个通道的 KH×KW，再排第二通道…直到第 IC 通道）。

步骤二：权重重排

将 \(N\) 个卷积核 OC×KH×KW×IC 展开为 OC × (KH×KW×IC) 的权重矩阵。

步骤三：矩阵乘法

输入矩阵 \((\text{OH} \times \text{OW}) \times (\text{KH} \times \text{KW} \times \text{IC})\) × 权重矩阵 \((\text{KH} \times \text{KW} \times \text{IC}) \times \text{OC}\) = 输出矩阵 \((\text{OH} \times \text{OW}) \times \text{OC}\)。

步骤四：Col2Im 还原

将输出矩阵还原为 \(N \times \text{OH} \times \text{OW} \times \text{OC}\) 三维张量。

Im2Col 优缺点总结

优点 缺点
充分利用 GEMM 库（BLAS/MKL） 内存膨胀约 (KH×KW−1) 倍
连续内存 + 缓存友好 大量数据复制开销
支持各种卷积参数 朴素实现性能受限
可与其他优化方法互补 大卷积核时内存消耗严重

19.3.3 空间组合优化算法

空间组合（Spatial Pack）是对 Im2Col 的进一步优化，通过分治法将大卷积拆分为多个小卷积。

常见优化方法

方法 原理 适用场景
分块卷积 大卷积核分解为多个小卷积核 大型图像/特征图
重叠数据块 通过重叠减少 Padding 需求 减少边缘计算
稀疏卷积 只对非零值进行卷积 稀疏特征图
权重共享 利用同一 Kernel 在不同位置复用 通用优化
张量化（Tensorization） 利用 SIMD 指令集 GPU 等并行硬件
循环展开 + 软件流水线 编译器优化循环结构 CPU 执行优化
内存复用 复用中间结果内存 减少内存分配开销

关键注意点：分块时需要额外拷贝相邻小张量的边界数据（用于 Padding）。拆分越小，局部性越好，但 Padding 引起的额外内存消耗越大。

19.4 Winograd 算法

19.4.1 核心思想

Winograd 算法的核心哲学：用更多的加法换更少的乘法。由于乘法指令的延迟远高于加法，减少乘法次数就能直接加速推理。

CVPR 2016 上 Lavin & Gray 提出的论文使其在推理引擎中广泛流行（MindSpore Lite、MNN 等均采用）。

一维卷积加速示例

以 \(F(2,3)\) 为例：输入 \(d = [d_0, d_1, d_2, d_3]^T\)，卷积核 \(g = [g_0, g_1, g_2]^T\)。

标准矩阵乘法：6 次乘法 + 4 次加法

Winograd 变换后：

  • \(m_1 = (d_0 - d_2) g_0\)
  • \(m_2 = (d_1 + d_2) \frac{g_0 + g_1 + g_2}{2}\)
  • \(m_3 = (d_2 - d_1) \frac{g_0 - g_1 + g_2}{2}\)
  • \(m_4 = (d_1 - d_3) g_2\)

4 次乘法 + 8 次加法。\(g\) 相关的计算可以在推理前预先完成（推理时卷积核是固定的），所以额外加法开销可控。

推广到二维卷积 \(F(2 \times 2, 3 \times 3)\)：

\[ Y = A^T \left[ (G g G^T) \odot (B^T d B) \right] A \]

其中 \(G\) 为卷积核变换矩阵，\(B\) 为输入变换矩阵，\(A\) 为输出变换矩阵，\(\odot\) 表示 Hadamard 积。

19.4.2 实现步骤

步骤一：卷积核变换 \(U = G g G^T\)

将 \(3 \times 3\) 卷积核 → 变换为 \(4 \times 4\) 矩阵 → 按位置重新排布 → 得到 16 个 IC×OC 的权重矩阵 \(U\)。

步骤二：输入数据变换 \(V = B^T d B\)

将输入切分为 \(4 \times 4\) 的小块（Tile） → \(B^T d B\) 变换 → 重新排布 → 得到 16 个 nr×IC 的输入矩阵 \(V\)。

步骤三：中间矩阵计算 \(M = \sum U \odot V\)

16 个 \(U \times V\) 矩阵乘 → 得到 16 个 nr×OC 的中间矩阵 \(M\) → 按位置重排为 \(4 \times 4\) 矩阵。

步骤四：输出变换 \(Y = A^T M A\)

将 \(4 \times 4\) 矩阵变换为 \(2 \times 2\) 输出 → 写回输出矩阵。

19.4.3 算法约束与缺点

优点 局限性
减少乘法次数，加速显著 仅适用于小卷积核（通常 3×3）
权重的变换可预计算 1×1、5×5、7×7 不使用
与分块策略可组合 加法次数增加 + 转换开销
分块越大，转换矩阵越大，精度损失加剧

工程实践：Runtime 层需要根据卷积参数决策使用哪个 Kernel（Im2Col、Winograd 还是直接卷积）。通常所有可固定的数据在预编译阶段计算好，避免在线重复计算。

19.5 QNNPACK 算法

19.5.1 前置知识回顾

Im2Col + GEMM 的两个缺陷

传统 GEMM 方法在 \(K\) 维度上拆分，每次计算核都需要加载和存储输出部分和：

  1. Im2Col 空间消耗大：输入图像和卷积核展开为二维矩阵，大量内存浪费
  2. GEMM 需要缓存大量行数据：大卷积核时即使分割也难避免

神经网络量化

QNNPACK 最初为量化推理设计——将 FP32 转换为低精度整数（FP16/INT8），提升计算速度和内存效率。但其核心算法（间接卷积）在非量化场景同样高效。

19.5.2 间接卷积算法

Marat Dukhan（Meta → Google）2019 年发表 The Indirect Convolution Algorithm，揭秘了 QNNPACK 的核心。

设计哲学

传统 GEMM QNNPACK
重新打包 A/B 矩阵以利用缓存层次 消除所有非计算必需的内存转换
面向科学计算（双精度大矩阵） 面向移动端量化推理（小矩阵适配 L1 缓存）
MR/NR 可达很大值 MR/NR ≤ 8（移动架构约束）

关键差异：QNNPACK 将整个 \(K\) 维在计算 Kernel 中处理完，消除了输出部分和的访存。

间接缓冲区（Indirection Buffer）

这是 QNNPACK 最精妙的设计——不复制输入数据到 Im2Col 缓冲区，而是建立一个指针数组指向输入行：

  • 每个输出像素位置 + 每个 Kernel 元素 → 一个指向输入行的指针
  • 超出输入范围的指针 → 指向显式零向量（处理 Padding）
  • 如果输入张量地址不变，间接缓冲区只需初始化一次，多次推理复用

间接缓冲区布局

间接缓冲区 = KH × KW × \(M\) 个指针，其中 \(M\) = OH × OW。计算时，\(M\) 个指针同时扫描输入，每次取出 \(M \times \text{IC}\) 的输入内存，配合权重完成 GEMM。

伪代码逻辑：

  for kw in 0..KW:
    从间接缓冲区取 M 个指针

19.5.3 与 Im2Col 算法对比

维度 Im2Col + GEMM QNNPACK 间接卷积
内存拷贝 需要将输入复制到 Im2Col 缓冲区 不需要，用指针索引
缓存效率 大卷积核缓存命中率低 间接缓冲区小，输入行可重用
额外开销 Im2Col 变换本身 加载指针的微弱开销
循环效率 单循环（KH×KW×IC 次） 双循环（略低于单循环）

QNNPACK 解决了卷积计算的三个问题：空间向量化、地址计算复杂、内存拷贝。代价是需要建立间接缓冲区和权重 Repacking。

19.6 推理内存布局

19.6.1 CPU 与 GPU 内存架构

CPU 内存架构：

  • 多级缓存（L1 → L2 → L3），容量递增、速度递减
  • 数据从主存加载到 Cache 时，邻近数据一并加载（空间局部性）
  • 缓存未命中时需等待主存操作完成

GPU 内存架构：

  • 同样多级缓存，以 SM（Streaming Multiprocessor）为单位
  • SM 内包含 Warp Scheduler、寄存器、L1 缓存、共享内存
  • SIMT 模式：一个 Warp 抓一把线程扔进 Core 计算
  • GPU 间通过 PCIe 桥或 NVLink 高速通道传输数据

19.6.2 NCHWX 内存格式

随着网络层数加深，特征图的 H/W 减小但 C（通道数）急剧增大（128、256 甚至更多）。为了充分利用矩阵计算单元，需要进行通道维度拆分。

格式 拆分粒度 典型场景
NCHW4 Channel / 4 ARM 处理器 4 通道 SIMD
NCHW32 Channel / 32 特定 AI 加速器
NCHW64 Channel / 64 高并行度 DSA 芯片

NCHWX 优势：

  • 适配 SIMT/SIMD，充分利用向量化指令
  • 对 Cache 更友好，减少缓存未命中
  • Kernel 层面易进行 Padding，减少边界分支判断

小结

第 19 章构建了推理引擎 Kernel 层优化的完整知识体系：

知识模块 核心要点
Kernel 层概述 CPU（NEON/AVX/Metal）+ GPU（CUDA/OpenCL/Vulkan）+ 算子库（cuDNN/oneDNN）；四大优化方向：算法、内存布局、汇编、调度
卷积计算原理 连续卷积 → 离散卷积（Toeplitz 矩阵）→ CNN 卷积（Padding/Stride/Channel/Kernel）；时间/空间复杂度公式
Im2Col 算法 卷积 → 矩阵乘，空间换时间，内存膨胀 (KH×KW−1) 倍；Spatial Pack 分治优化
Winograd 算法 加法换乘法（6→4 次），\(Y = A^T[(GgG^T) \odot (B^TdB)]A\)；仅适用小卷积核（3×3）
QNNPACK 间接缓冲区替换 Im2Col 拷贝，PDOT 微内核，适配 L1 缓存的轻量设计
内存布局 NCHW vs NHWC vs NCHWX，通道拆分适配硬件并行度

核心启示：Kernel 优化的演进史本质上是对"计算 vs 访存 vs 空间"三元悖论的持续平衡。从 Im2Col（牺牲空间换访存便利）到 Winograd（牺牲通用性换计算减少）再到 QNNPACK（牺牲灵活性换零拷贝），每一代算法都在寻找更优的平衡点。理解这些算法的设计取舍，才能真正领会推理引擎 Kernel 层的设计精髓。
