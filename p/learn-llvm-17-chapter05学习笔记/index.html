第 5 章 高级语言结构生成的 IR - 学习笔记

本文为《Learn LLVM 17》第 5 章的学习笔记，主要介绍如何将聚合数据类型（数组、结构体、指针）和面向对象结构（类、虚函数）转换为 LLVM IR，以及应用程序二进制接口（ABI）相关知识。

1. 本章概览

1.1 学习目标

主题 内容
聚合类型 数组、结构体、指针的 IR 表示与访问
ABI 应用程序二进制接口，参数传递规则
OOP 类、虚函数表、单继承、多重继承、接口

2. 处理数组、结构体和指针

2.1 LLVM IR 中的聚合类型

数组 (Array)

  • 语法：[N x T]，N 为元素个数，T 为元素类型
  • 特点：静态大小，元素数量恒定
  • 示例：ARRAY [10] OF INTEGER → [10 x i64]
1
2
3
4
5
6
7

; 全局数组声明（等价于 C 的 long arr[8]）
@arr = global [8 x i64] zeroinitializer

define i64 @second() {
  %1 = load i64, ptr getelementptr inbounds ([8 x i64], ptr @arr, i64 0, i64 1)
  ret i64 %1
}

结构体 (Struct)

  • 语法：{ T1, T2, ... }，按顺序列出成员类型
  • 访问：使用数字索引，从 0 开始
  • 示例：RECORD x: REAL; color: INTEGER; y: REAL; END → { float, i64, float }
1
2

; 打包结构体（1 字节对齐，无填充）
<{ float, i64, float }>

extractvalue / insertvalue

  • extractvalue：从聚合值中提取元素（索引必须为常量）
  • insertvalue：将元素插入聚合值
1
2
3

%el2 = extractvalue [10 x i64] %x, 1
%xnew = insertvalue [10 x i64] %x, i64 %el2, 0
%color = extractvalue { float, float, i64 } %pt, 2

限制：索引必须为常量，数组无法动态索引，一般需借助指针和 getelementptr。

2.2 getelementptr 指令

地址计算的核心指令，不涉及内存存取。

1
2
3
4

; 语法：getelementptr inbounds <基类型>, <基指针>, <索引列表>
; 访问 arr[1]：先解引用 (i64 0)，再索引 (i64 1)
%ptr = getelementptr inbounds [8 x i64], ptr @arr, i64 0, i64 1
%val = load i64, ptr %ptr
  • 基类型：指针指向的完整类型（如 [8 x i64]）
  • 基指针：全局变量/局部变量等
  • 索引：第一个索引通常为 0（解引用），后续为元素/字段索引
  • 结构体：字段索引必须为 i32 常量

// Chapter05/tinylang/lib/CodeGen/CGModule.cpp:26-69

llvm::Type *CGModule::convertType(TypeDeclaration *Ty) {
  if (llvm::Type *T = TypeCache[Ty])
    return T;

  if (llvm::isa<PervasiveTypeDeclaration>(Ty)) {
    // INTEGER -> i64, BOOLEAN -> i1
  } else if (auto *ArrayTy = llvm::dyn_cast<ArrayTypeDeclaration>(Ty)) {
    llvm::Type *Component = convertType(ArrayTy->getType());
    Expr *Nums = ArrayTy->getNums();
    uint64_t NumElements = llvm::cast<IntegerLiteral>(Nums)->getValue().getZExtValue();
    llvm::Type *T = llvm::ArrayType::get(Component, NumElements);
    return TypeCache[Ty] = T;
  } else if (auto *RecordTy = llvm::dyn_cast<RecordTypeDeclaration>(Ty)) {
    llvm::SmallVector<llvm::Type *, 4> Elements;
    for (const auto &F : RecordTy->getFields()) {
      Elements.push_back(convertType(F.getType()));
    }
    llvm::Type *T = llvm::StructType::create(Elements, RecordTy->getName(), false);
    return TypeCache[Ty] = T;
  }
  llvm::report_fatal_error("Unsupported type");
}

注意：Chapter05 的 tinylang 未实现 PointerTypeDeclaration 的 convertType，仅 AST 支持指针选择器。

// 读取变量基址/值
llvm::Value *Val = readVariable(Curr, Decl);

auto &Selectors = Var->getSelectors();
for (auto I = Selectors.begin(); I != Selectors.end(); ) {
  if (auto *IdxSel = llvm::dyn_cast<IndexSelector>(*I)) {
    llvm::SmallVector<llvm::Value *, 4> IdxList;
    while (I != E && llvm::dyn_cast<IndexSelector>(*I)) {
      IdxList.push_back(emitExpr(Sel->getIndex()));
      ++I;
    }
    Val = Builder.CreateInBoundsGEP(Val->getType(), Val, IdxList);
    Val = Builder.CreateLoad(Val->getType(), Val);
  } else if (auto *FieldSel = llvm::dyn_cast<FieldSelector>(*I)) {
    llvm::SmallVector<llvm::Value *, 4> IdxList;
    while (I != E && llvm::dyn_cast<FieldSelector>(*I)) {
      IdxList.push_back(llvm::ConstantInt::get(CGM.Int64Ty, Sel->getIndex()));
      ++I;
    }
    Val = Builder.CreateInBoundsGEP(Val->getType(), Val, IdxList);
    Val = Builder.CreateLoad(Val->getType(), Val);
  } else if (auto *DerefSel = llvm::dyn_cast<DereferenceSelector>(*I)) {
    Val = Builder.CreateLoad(Val->getType(), Val);
    ++I;
  }
}
return Val;

// Chapter05/tinylang/lib/CodeGen/CGProcedure.cpp:365-398

if (Selectors.empty()) {
  writeVariable(Curr, Desig->getDecl(), Val);
} else {
  llvm::SmallVector<llvm::Value *, 4> IdxList;
  IdxList.push_back(llvm::ConstantInt::get(CGM.Int32Ty, 0));  // 首索引
  auto *Base = readVariable(Curr, Desig->getDecl(), false);   // 不加载，取地址

  for (auto I = Selectors.begin(); I != Selectors.end(); ++I) {
    if (auto *IdxSel = llvm::dyn_cast<IndexSelector>(*I))
      IdxList.push_back(emitExpr(IdxSel->getIndex()));
    else if (auto *FieldSel = llvm::dyn_cast<FieldSelector>(*I))
      IdxList.push_back(llvm::ConstantInt::get(CGM.Int32Ty, FieldSel->getIndex()));
  }

  Base = Builder.CreateInBoundsGEP(mapType(Desig->getDecl()), Base, IdxList);
  Builder.CreateStore(Val, Base);
}

注意：readVariable(..., false) 表示不执行 load，直接返回地址（用于 VAR 参数和全局变量）。

// Chapter05/tinylang/lib/Sema/Sema.cpp:479-522

void Sema::actOnIndexSelector(Expr *Desig, SMLoc Loc, Expr *E) {
  if (auto *D = dyn_cast<Designator>(Desig)) {
    if (auto *Ty = dyn_cast<ArrayTypeDeclaration>(D->getType()))
      D->addSelector(new IndexSelector(E, Ty->getType()));
  }
}

void Sema::actOnFieldSelector(Expr *Desig, SMLoc Loc, StringRef Name) {
  if (auto *D = dyn_cast<Designator>(Desig)) {
    if (auto *R = dyn_cast<RecordTypeDeclaration>(D->getType())) {
      uint32_t Index = 0;
      for (const auto &F : R->getFields()) {
        if (F.getName() == Name) {
          D->addSelector(new FieldSelector(Index, Name, F.getType()));
          return;
        }
        ++Index;
      }
    }
  }
}

void Sema::actOnDereferenceSelector(Expr *Desig, SMLoc Loc) {
  if (auto *D = dyn_cast<Designator>(Desig)) {
    if (auto *Ty = dyn_cast<PointerTypeDeclaration>(D->getType()))
      D->addSelector(new DereferenceSelector(Ty->getType()));
  }
}

3. 应用程序二进制接口 (ABI)

3.1 为什么需要 ABI

不同平台对以下问题有不同规则：

  • 参数用哪些寄存器传递？
  • 聚合类型如何传递（按值/按引用/拆分）？
  • 返回值如何传递？
  • 浮点/向量是否用专用寄存器？

示例：{ float, float } 复数类型

  • C99 _Complex：可能在两个浮点寄存器中传递
  • 用户定义 struct { float re, im; }：可能按指针间接传递

LLVM 需要额外信息才能生成符合 ABI 的调用代码。

3.2 参数属性

属性 含义
inreg 参数在寄存器中传递
byval 按值传递，形参为指针，会生成隐藏副本
zeroext 零扩展
signext 符号扩展
sret 形参为返回聚合类型的指针 
1

// 添加属性
Arg->addAttr(llvm::Attribute::InReg);

// 多个属性
llvm::AttrBuilder Attr(Fn->getContext());
Attr.addDereferenceableAttr(Sz);
Attr.addAttribute(llvm::Attribute::NoCapture);
Arg->addAttrs(Attr);

3.3 类型重写

  • 拆分参数：如将 { float, float } 拆成两个 float
  • 表示转换：如通过整数寄存器传浮点，用 bitcast 转换
1

%intconv = bitcast float %fp to i32

bitcast 要求两种类型大小相同。

  • Clang 实现：https://github.com/llvm/llvm-project/blob/main/clang/lib/CodeGen/TargetInfo.cpp

4.1 单继承

4.1.1 Oberon-2 示例

TYPE Shape = RECORD
  color: INTEGER;
  PROCEDURE (VAR s: Shape) GetColor(): INTEGER;
  PROCEDURE (VAR s: Shape) Area(): REAL;  (* 抽象 *)
END;

TYPE Circle = RECORD (Shape)
  radius: REAL;
  PROCEDURE (VAR s: Circle) Area(): REAL;  (* 重写 *)
END;

4.1.2 内存布局

  • 基类数据在前，子类数据在后
  • 每个对象第一个字段为 vtable 指针
1
2

@Shape = type { ptr, i64 }           ; vtable, color
@Circle = type { ptr, i64, float }    ; vtable, color, radius

4.1.3 虚函数表 (vtable) 

1

@ShapeVTable = constant { ptr, ptr } { ptr @GetColor, ptr @Area }
  • 每个类一个 vtable
  • 方法按声明顺序排列

4.1.4 动态调用步骤

  1. GEP 取对象首字段 → vtable 指针
  2. load vtable
  3. GEP 取方法在 vtable 中的偏移
  4. load 函数指针
  5. 通过指针间接 call
1
2
3
4
5

%ptrToShapeObj = load ptr, ...
%vtable = load ptr, ptr %ptrToShapeObj, align 8
%offsetToArea = getelementptr inbounds ptr, ptr %vtable, i64 1
%ptrToAreaFunction = load ptr, ptr %offsetToArea, align 8
%result = call float %ptrToAreaFunction(ptr %ptrToShapeObj)

4.1.5 方法 = 函数 + this

  • Oberon-2：显式接收者 (VAR s: Shape)
  • C++：隐式 this
  • 实现：将对象指针作为第一个参数传递

4.2 运行时类型测试

简单实现：比较 vtable 指针

  • shape IS Circle：仅当 shape 的 vtable 等于 Circle 的 vtable 时为真
  • 若 shape 实际为 Ellipse（Circle 的子类），则返回 false

扩展 vtable：在 vtable 中增加指向基类 vtable 的指针，可沿继承链向上比较。

高效方案：对类层次做深度优先编号，类型测试变为 O(1) 的区间/数字比较（类似 LLVM 的 llvm.type.test）。

4.3 接口扩展

  • 接口 = 抽象方法集合
  • 不同接口的方法在 vtable 中顺序可能不同
  • 方案：为每个接口维护单独的接口 vtable，或使用哈希表将接口 ID 映射到 vtable

4.4 多重继承

  • 布局：先放第一个基类（含 vtable），再放第二个基类（含 vtable），最后是派生类成员
  • 同一对象可有多个“合法”指针（指向不同基类子对象）
  • 调用虚方法时可能需要 this 指针调整：根据调用来源（哪个基类）调整 this，再调用

vtable 扩展：每个槽可存 (函数指针, this 调整偏移)。

5.2 与第 4 章的主要差异

特性 第 4 章 第 5 章
类型 INTEGER, BOOLEAN + Array, Record
变量访问 简单变量 + 索引、字段、解引用
局部变量 SSA 聚合类型用 alloca
选择器 无 IndexSelector, FieldSelector, DereferenceSelector

5.3 未实现部分

  • ProcedureCallStatement：emitStmt 中直接 report_fatal_error
  • PointerTypeDeclaration：AST 有，convertType 未处理
  • ABI 属性：仅 VAR 参数有 Dereferenceable、NoCapture
  • 类与虚函数：本章为概念说明，tinylang 未实现

6. 总结

本章覆盖：

  1. 聚合类型：数组 [N x T]、结构体 { T1, T2, ... }，用 GEP + load/store 访问
  2. 选择器：索引 arr[i]、字段 rec.field、解引用 ptr^ 的 AST 与代码生成
  3. ABI：参数属性（inreg、byval、sret 等）与类型重写
  4. OOP：vtable 布局、动态调用、单继承与多重继承、接口与类型测试

学习了以上这些内容后，可以为支持聚合类型和 OOP 的语言设计并实现 LLVM IR 代码生成器。
