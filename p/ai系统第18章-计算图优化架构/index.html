AI系统：第18章 计算图优化架构

《AI系统：原理与架构》学习笔记 — 深入理解推理引擎的图优化模块：从四种冗余到常量折叠、节点消除、算子融合、算子替换、FlashAttention 等核心技术

4 minute read

本章概览

如果说前面的章节介绍了推理引擎的整体架构，那么第 18 章则将焦点对准推理引擎中最核心的优化环节——计算图优化（Graph Optimization）。

计算图是一种表示和执行数学运算的数据结构。在深度学习中，模型的训练和推理过程通常被表示成一个复杂的计算图，其中节点代表运算操作，边代表张量在操作之间的流动。推理引擎中的图优化模块负责实现计算图中的各种优化操作，包括算子融合、布局转换、算子替换和内存优化等，目标是提高计算效率和减少内存占用。

本章从离线优化模块面临的四类冗余出发，系统梳理了基础优化与扩展优化两大范式，然后深入到常量折叠、冗余节点消除、算子融合/替换/前移等具体技术，最后重点介绍了 FlashAttention——一种在大模型时代至关重要的注意力计算优化算法。

核心洞察：图优化的本质是对"冗余"的识别与消除。无论是结构性冗余、精度冗余、算法冗余还是读写冗余，图优化模块都在做一件事——用更少的计算和更少的内存，完成同样的推理任务。

18.1 引言：挑战、方案与优化方式

18.1.1 离线模块的四大挑战

推理引擎的离线优化模块面临四种核心冗余：

冗余类型 说明 优化方向
结构冗余 无效计算节点、重复计算子图、相同结构模块 算子融合、算子替换、常量折叠
精度冗余 FP32 范围宽但存在冗余，可压缩到 FP16/INT8 量化、稀疏化、低秩近似
算法冗余 算子/Kernel 实现算法本身存在计算冗余 Kernel 泛化性提升、多尺度 Kernel
读写冗余 重复读写内存、内存访问不连续 数据排布优化、内存池管理

18.1.2 推理 vs 训练中的图优化差异

AI 编译器的前端优化已经在训练场景中广泛应用，但推理引擎的图优化有其独特之处：

  • 训练场景：实验时间相对宽松，可引入较多 JIT 编译等高级手段
  • 推理场景：更多采用预先写好的模板进行图转换，基于先验知识实现优化，追求极致的确定性和低延迟

这种"模板驱动"的方式在特定场景能带来巨大计算收益，但缺点是需要根据先验知识手动设计优化规则。

18.1.3 图优化的方式

图优化分为**基础优化（Basic）和扩展优化（Extended）**两大类：

五大基础优化方式：

  1. 常量折叠（Constant Folding）：编译时预先计算常量节点，替换为结果值
  2. 冗余节点消除（Redundant Node Elimination）：删除不参与计算的算子
  3. 有限数量的算子融合（Operator Fusion）：合并相邻算子减少访存
  4. 扩展优化（Extended）：针对特定硬件（如 CUDA）的特殊优化
  5. Layout & Memory 优化：布局转换与内存分配优化

三个阶段的分工：

  • 预优化（Pre Optimize）：Basic 优化方式
  • 优化（Optimize）：Basic + Extended 混合
  • 后优化（Post Optimize）：Extended 优化 + Layout & Memory

18.2 离线图优化技术

18.2.1 常量折叠

常量折叠是编译器最经典的优化技术之一——在编译时计算出常量表达式的结果，消除运行时计算。

基本原理：

# 优化前
x = 5
y = 10
z = x * y

# 常量折叠后
z = 50

三种具体折叠形式：

1. Const 折叠

如果某 Op 的所有输入都是常量，先计算好结果替代该 Op，避免每次推理重复计算。

图18.2.1：Op1 的两个输入均为常量，离线阶段预先计算后作为新常量直接输入 Op2。

2. ExpandDims 折叠

当 ExpandDims 的指定维度输入是常量时，将该维度以参数形式折叠到算子内部，减少一个算子。

3. Binary 折叠

Binary Op 的第二个输入是标量常量时，将该标量以参数形式折叠到 Binary Op 的属性中。

18.2.2 冗余节点消除

冗余节点消除在不改变图形结构的前提下删除无意义节点，分为五种形态：

1. Op 本身无意义

消除情况 具体描述
冗余算子删除 去掉 Identity、NoOp、Print、Assert、StopGradient、Split 等
Dropout 删除 删除推理阶段不再需要的 Dropout 节点
Cast 删除 Cast 转换前后数据类型相同时删除
Concat 删除 Concat 只有一个输入 Tensor 时删除

图18.2.4：冗余算子消除的三种情况——直接连接、切分为两个子图、轮循删除直到输入有意义为止。

2. Op 参数无意义

场景 消除条件
Cast 消除 参数 Src = Dst 时无意义
Slice 消除 index_start=0 或 index_end=channel-1 时无意义
Expand 消除 输出 Shape = 输入 Shape 时无意义
Pooling 消除 滑窗为 1×1 时无意义

3. Op 位置无意义

某些算子在计算图中特定位置会变得多余：

图18.2.6：Remove Output Cast、UnSqueeze Elimination、Orphaned Elimination、Reshape after Global Pooling 等场景。

图18.2.7：Flatten after Linear Elimination、重复的 Reshape 或 Cast 均可删除。

4. Op 前后反义

前后两个相邻 Op 语义相反时，两者均可删除：

  • Squeeze ↔ ExpandDims：压缩与扩展维度互为逆向
  • Inverse Cast：A→B→A 等同于无操作
  • Quant ↔ Dequant：量化与反量化抵消
  • Concat ↔ Slice：合并后拆分可简化为直接索引

5. 公共子图优化（CSE）

当模型中出现类型、参数和输入均相同的公共子图时，只需计算一次，其他通过赋值复用结果。

实现思路：通过一个 MAP 表记录已处理过的同类型 Op，匹配到相同输入和参数的 Op 即为公共子表达式。

18.2.3 算子融合

算子融合是深度学习中减少内存访问的核心技术。以 y = ReLU(Conv(x)) 为例，分开计算需要两次访存（先写 Conv 结果，再读回做 ReLU），融合后只需一次访存。

示例一：以 Conv 为中心的融合

1) Conv + BN + Act

将 BN 的 \( \gamma \) 和 \( \beta \) 参数融合到 Conv 的权重和偏置中：

Python 示例：

def fuse_conv_bn(conv, bn):
    w = conv.weight
    mean = bn.running_mean
    var_sqrt = torch.sqrt(bn.running_var + bn.eps)
    gamma = bn.weight
    beta = bn.bias
    w_prime = gamma / var_sqrt * w
    b_prime = beta - gamma / var_sqrt * mean
    conv.weight.data = w_prime
    conv.bias.data = b_prime
    return conv

2) Conv + Bias + Add：Add 的值直接合并到 Bias 中，b' = b + a

3) Conv + Scale + Act：Scale 参数 α 融合到权重，W' = αW, b' = αb

4) Conv + MatMul + Act：与 Scale 融合原理相同

示例二：更多融合模式

融合模式 优化方式
MatMul + Add 使用 GEMM 替代
MatMul + Add/Scale/Div 通过数学交换律持续向后融合（可能牺牲精度）
Mean + Add 使用 Layer Norm 代替
Batch Norm + Scale Scale 的 s 和 b 直接融入 BN
MatMul + Batch Norm 类似 Conv + BN
Conv + ReLU / ReLU6 / Act 计算后立即激活，减少一次访存

注意：MatMul 融合虽然性能提升，但 Scale 通常 < 1，融合后权重数值进一步降低，可能导致精度下降。

18.2.4 算子替换

算子替换通过合并同类项、提取公因式等数学方法，将原算子替换为功能相同但更高效的算子。

一换一替换（单一算子替换）

  • MatMul → Conv2D：利用框架对 Conv 的深度优化
  • Linear → Conv2D：全连接层转为 1×1 Conv
  • Batch Normal → Scale：等价转换，计算量更少
  • pReLU → Leaky ReLU：不影响性能前提下聚焦有限算法
  • Conv → Linear（Global Pooling 之后）：简化后续处理

一换多替换（组合算子替换）

  • Shuffle Channel：通过 Reshape + Permute 组合实现
  • Pad Replace：老版 ONNX pad-2 的参数转输入形式
  • Group Conv：通过 Slice + Conv 组合实现
  • ShapeN：通过多个 Shape 操作组合实现

18.2.5 算子前移

算子前移将固定不变的计算提前执行并保存结果，避免每次推理重复计算。典型应用：

  • Slice and Mul：利用交换律调整算子顺序
  • Bit shift and Reduce Sum：通过算术简化减少数据传输

算子前移需要谨慎考虑计算顺序和数据依赖性，不能随意将计算提前。

18.3 其他计算图优化

18.3.1 融合算子替换

某些复杂算子在框架中没有直接实现，而是通过基本算子组合实现，导致效率降低。融合算子替换将这些组合转为单一高效算子：

融合类型 说明
Fuse Layer Norm 组合实现的 Norm → 单一 Op
Fuse PReLU 组合实现的 PReLU → 单一 Op
Fuse MatMul Transpose Transpose + MatMul → 带转置参数的 MatMul
Fuse Binary Eltwise Add + Mul 融合为带系数的 Eltwise Sum
Fuse Reduction → Global Pooling 两次 reduction mean → 一次 global_mean_pooling

18.3.2 FlashAttention —— 改变游戏规则

FlashAttention 是本章最具深度的小节，也是当前大模型时代最著名的系统优化技术之一。

问题背景

Transformer 的核心是 Attention 机制，其计算复杂度为 \( O(N^2) \)，这导致：

  • 长序列无法处理：早期 GPT 只支持 2k~4k token
  • 二次方增长：序列长度翻倍，资源消耗变为 4 倍
  • 显存瓶颈：\( S \) 和 \( P \) 矩阵需要 \( O(N^2) \) 的存储空间

GPU 存储架构的关键洞察

A100 GPU 有两个存储层级：

  • HBM：4080 GB，带宽 1.52.0 TB/s（慢但大）
  • SRAM：每 108 个 SM 有 192 KB，带宽约 19 TB/s（快但小）

SRAM 的访问速度是 HBM 的约 10 倍！标准 Attention 的问题在于：每个 Kernel 都从 HBM 读取数据、计算、再写回 HBM，产生了大量冗余的 HBM 访问。

FlashAttention 的两大核心技术

图18.3.2：FlashAttention 算法伪代码 — 核心在于 Tiling（分块计算）+ Recomputation（反向重算）。

1. Tiling（平铺）

将 \( N \times N \) 的 softmax/scores 矩阵分块处理，每个 Block 在 SRAM 中完成计算：

  • 外循环：加载 \( K \)、\( V \) 的 Block 到 SRAM
  • 内循环：加载 \( Q \) 的 Block 到 SRAM
  • 在 SRAM 上完成 Block S 的计算

2. 分块 Safe Softmax

标准 Attention 需要全局最大值来稳定 softmax 计算，分块后无法拿到全局信息。FlashAttention 通过数学证明：分块 softmax 与全局 softmax 在数学上等价，通过维护 \( m \)（行最大值）和 \( \ell \)（累积分母）的递推更新来实现。

关键公式（简化版）：

  • 计算当前块 \( S_{ij} \) → 块最大值 \( m_{ij} \) → 块 softmax 分子 \( P_{ij} \) → 块累积和 \( \ell_{ij} \)
  • 与上一轮结果合并：\( m^{new} = \max(m, m_{ij}) \)，\( \ell^{new} = e^{m - m^{new}} \cdot \ell + e^{m_{ij} - m^{new}} \cdot \ell_{ij} \)
  • 最终：\( O_i = \text{diag}(\ell^{new})^{-1} \cdot \left[ \text{diag}(\ell) \cdot e^{m - m^{new}} \cdot O_i + e^{m_{ij} - m^{new}} \cdot P_{ij} \cdot V_j \right] \)

3. Recomputation（重计算）

前向过程只保存 \( O \)、\( \ell \)、\( m \) 到 HBM，反向传播时重新计算 \( S \) 和 \( P \)，以计算换存储。

性能分析

FlashAttention 的 HBM 访问次数从标准 Attention 的 \( O(N \times d + N^2) \) 降低到 \( O(N^2 \times d^2 / M) \)，其中 \( M \) 是 SRAM 大小。三个 Kernel 合并为一个，中间计算完全基于 SRAM。

18.3.3 Layout & Memory 优化

当上一层和下一层的算子输入输出格式不同时，需要插入特定算子进行数据节点转换。同时，内存优化通过以下策略减少内存占用：

Inplace Operation（原地操作）：如果一块内存不再需要，且下一个操作是 Element-Wise（如加法、乘法），直接在原内存上计算覆盖，节省内存分配和回收开销。

Memory Sharing（内存共享）：当两个数据大小相同且前一个不再需要时，后一个直接覆盖前一个的内存。

小结与思考

第 18 章构建了推理引擎图优化的完整知识体系：

  • 为什么需要图优化？ 神经网络模型存在结构、精度、算法和读写四类冗余。图优化通过自动化的方式消除这些冗余，降低运行成本和内存占用。

  • 图优化的核心手段有哪些？

    • 常量折叠：编译时计算，消除运行时开销
    • 冗余节点消除：五种形态（无意义 Op、无意义参数、无意义位置、前后反义、公共子图）精简图结构
    • 算子融合：将多个算子合并为一个，减少内存访问次数
    • 算子替换：用高效算子替代低效实现
    • 算子前移：将固定计算提前执行

  • FlashAttention 为什么重要？ 它完美诠释了"利用更高速的存储层减少低速存储访问"这一系统优化的核心思想。通过 Tiling + 分块 Softmax + Recomputation 三项技术，让 Attention 计算中的中间结果完全驻留在 SRAM 中，大幅降低了 HBM 访问次数。这是大模型能支持更长序列的关键技术突破。

  • 图优化三阶段：预优化（Basic）→ 优化（Basic + Extended）→ 后优化（Extended + Layout/Memory），层层递进。

如果说 Transformer 是 AI 的"发动机"，那 FlashAttention 就是让这个发动机能持续高速运转的"涡轮增压器"。而图优化整体，就像是对发动机的全面调校——从减少零件冗余（节点消除）、精简管道（算子融合）、到升级点火系统（常量折叠）——让每一滴燃油（计算资源）都发挥最大价值。
